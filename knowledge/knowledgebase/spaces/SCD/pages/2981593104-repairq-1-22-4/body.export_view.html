Settings and Operations

  • Resolved an issue where saving settings on the Account → General Settings screen without configuring all currency fields resulted in missing values for Coins and Bills in the Reconciliation Report.

  • Resolved an issue where exports using (Queued Tasks) would fail and return an error. Users can now successfully export their ticket queue without encountering errors.

  • Resolved an issue where staff members without a location assignment were unsearchable in the Staff List. Users can now search for staff without location assignments by clearing the Location filter when searching the Staff List.

  • Resolved an issue where staff members without a location assignment were excluded from inactivity tracking and the auto-deactivation processes. These users are now correctly identified as inactive and included in the auto-deactivation tool.

Appointments

  • Resolved an issue where Appointment concurrency limits were not enforced when creating appointments within the UI while Appointment Windows were enabled.

  • Resolved an issue where the Day View of the Appointment Calendar only displayed the appointment time and customer name, even when additional fields (Assignee Name, Device Name, etc.) were configured to appear. These fields are now correctly displayed in the appointment blocks on the Day view, ensuring consistency with the Week and Month views.

Custom Forms and Automated Notifications

  • Resolved an issue where attempting to save a new Custom Form Field resulted in an error message: "Template type cannot be blank."

  • Updated the Automated Notification creation form by marking the Ticket Actions field as required with an asterisk (*).

Inventory Management

  • Resolved an issue where users encountered an Error 500 when filtering items by keywords on the Catalog page. Searching for items using keywords now functions as expected without errors.

  • Resolved an issue where creating an RMA batch caused the modal to remain in a loading state without generating any action. This process now completes as expected.

Tickets and Quotes

  • Resolved an issue where the "Add Stock" modal failed to appear when adding inventoried, in-stock items using the Catalog Quick Create feature on a ticket.

  • Resolved an issue where updates to the Logistics section of a repair ticket were not saved if a user made edits to customer information after entering logistics details. Logistics settings are now retained after customer edits.

  • Resolved an issue where editing serialized items on a Trade-in ticket caused the Serial Number field to be removed. This issue affected tickets with multiple items, requiring users to re-enter serial numbers after saving changes. Items now retain their serial numbers correctly after being edited

  • Resolved an issue where customers with net billing terms encountered a "Customer not found for invoicing" error when creating an invoice. Invoices for customers with net billing terms can now be created without any errors.

  • Resolved an issue preventing users from closing quotes without requiring payment. Users can now resume moving quotes to Closed status anytime, restoring the previous functionality.
