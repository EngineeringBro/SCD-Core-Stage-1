Importing staff is an efficient way to add or edit staff members in bulk, saving time and reducing manual entry. Import capabilities are restricted to users with Admin and Manager roles. This ensures that only authorized personnel handle critical data operations, maintaining data integrity and security.

RepairQ accepts Excel (.xlsx), Comma-Separated Values (.csv), and Tab-Separated Values (.txt). We recommend using .xlsx files for the best upload experience.

Note: When importing staff updates, any user updated with this import process will have their password reset, so notify the staff members after completing the import.

Download the Staff Import Template

On the Navigation Bar, go to Staff → View Staff. At the bottom of the Staff List, click Download Import Template to download an Excel file with your staff members.

Note: The information on the import template will reflect your filtered list of active users.

Staff Import Template

New Staff Records

Imports are matched by the column name when creating or updating users. Any unrecognized column headers will be ignored. If no matches are found for a record in your import file, RepairQ will create a new staff record from the import row.

Note: After importing your staff list, have your staff members use the Password Reset feature on the login screen to assign a new PIN/Passcode. Alternatively, a manager or an admin can reset their password manually by editing their staff record.

Required Fields

  • First Name - The first name of the staff member/user

  • Last Name - The last name of the staff member/user

  • Email - The contact email address of the user

  • Username - A unique identifier used to access the environment

Recommended Optional Fields

The following fields are not required to create a new staff member, however, leaving these blank will require updates to the user profile before they can successfully log into the environment:

  • Active - (yes/no) This will determine whether the user is active or inactive in the environment. Inactive staff profiles are hidden and are retrieved by filtering your Staff List for Inactive users.

  • IPs - A trusted list of IP addresses allowed to access the environment. This helps to keep the system secure by blocking access from unknown, unauthorized, or potentially harmful IP addresses.

    • Trusted IP data is recognized in “Name:IP Address” format. For example, “Whitelist Name:8.8.8.8”

    • To enable access from any location, use “Anywhere:*.*.*.*”

  • Role Type - There are three levels of staff access in your environment.

    • User - Default access to locations with an assigned role for basic operations, customers, inventory, and tickets. Restricts the user from administrative access.

    • Admin - Full permission for all Company and Location settings, operations, permissions, and data.

    • Accountant - Read-only access at all locations for viewing tickets and standard reports, with report export access.

  • Location Roles - Assign location-based roles for the user. These include store operations, store settings, and store-level decision-making.

    • Roles are recognized in “Location Short Name:Role” format. If assigning multiple roles for the same location, they are comma-separated, with no spaces.

      • Single Location, Single Role - “RepairQ Tulsa:assistant_manager”

      • Single Location, Multiple Roles - “RepairQ Tulsa:billing_manager,sales_manager”

    • If a user has roles at multiple locations, these are separated by the pipe symbol ( | ) with no spaces.

      • Multiple Locations, Single Role - “RepairQ Tulsa:billing_manager|RepairQ Bville:billing_manager”

      • Multiple Locations, Multiple Roles - “RepairQ Tulsa:billing_manager,sales_manager|RepairQ Bville:billing_manager,sales_manager”

    • For a detailed list of responsibilities for each role, see Roles and Permissions.

      • Invoicing = billing_manager

      • Technician = technical_manager

      • Sales = sales_manager

      • Shift Lead = shift_lead

      • Inventory Manager = inventory_manager

      • Manager = assistant_manager

      • Location Admin = manager

Optional Fields

These fields are optional, but filling them out will assign preferred contact methods, set discount limits, and manage timecards and labor costs.

  • Contact Method - The user’s preferred contact method

  • Primary Phone - The user’s primary phone number

  • Alternate Phone - The user’s alternate phone number

  • SMS Phone - The user’s mobile number (required for automated notifications)

  • SMS Carrier - The user’s mobile carrier (required for automated notifications)

    • Carrier assignments are managed in Company Settings → Catalog → Carriers.

  • Hourly Pay Rate - This is used for tracking labor in your Profitability Reports. (no symbol required)

  • Discount Limit Type - (“amount” or “percentage”) Set a discount limit based on a set amount or percentage.

  • Discount Limit: 0-100 - Assign a numerical amount based on the Limit Type. (no symbol required)

  • Timesheet Required - (yes/no) Choose whether the user must clock in to record hours worked.

Update Staff Records

If a record in your import file matches an existing record, it will be updated with the new data from your spreadsheet. The following fields identify matches for your records:

  • Username

  • Email

Import the Staff Template

When ready to upload your import, navigate to Staff → View Staff. At the bottom of the Staff List, click Choose File and select your file from the file explorer. Next, click Import. RepairQ will begin processing your import file. You will be notified via email when the import is complete, and you can check the Import Queue for any additional details.

Import Rules

Columns are matched on the header name, so you must have the correct header names (provided in the template) to import the column values successfully. Any unrecognized column headers will be ignored; only the values you wish to add or update need to be present in the import file. If no matches are found for a record in your import file, RepairQ will create a new User Profile record from the import row.

Exporting Staff Lists

You can export a list of Staff members for your records or to make changes to staff in bulk.

On the Navigation Bar, go to Staff → View Staff. Use the filters on the left column to narrow the list based on your requirements (Name, Location, Status, User Type, Role), then click Apply Results. If you are satisfied with the results on the list, click Download Import Template at the bottom of the page. The file will include all the information based on your filtered results.
