Released 08/15/2022 at 11:30pm CT

Minor Changes

  • Cleaned up the ticket label format, so that certain text fields fit better when printing

  • Setting up the Square integration no longer pulls in locations that are inactive

  • Improved the usability of the "Appointment Statuses" page

Bug Fixes

  • Checking or unchecking the "Skip customer receipt preference" option for a Square terminal now saves properly

  • Resolved an issue where customer profile information was not being saved properly

  • Adding backordered parts to a refurbishment ticket is now supported and no longer returns an error

  • When navigating between "Notes" and "Custom Forms" within a new ticket, custom forms no longer disappear

  • Updating a trade-in layaway ticket no longer retroactively updates related items that had already been refurbished and sold back to "Instock"

  • Serialized backorder items will no longer prevent a ticket with other items attached from being saved

  • Customer referrals now reflect properly

  • Fixed an issue where submitting inventory counts would return an error about "Location ID" and "Created By" fields being blank

  • Resolved some email and text notifications errors, which would cause the notifications to get stuck in "New" status

  • Diagnostic notes on a Return Repair ticket no longer overwrite diagnostic notes on the original repair ticket

  • Fixed an issue that was preventing a user from starting a return from the ticket list under the customer profile

  • Inventory for inactive locations will no longer appear in search inventory results
