This guide walks you through automating the process of sending customer data from RepairQ to your Mailchimp account. Setting up this automation streamlines your marketing efforts and engages with customers more effectively. For example, after completing a repair or purchase, customers can be automatically added to your Mailchimp list, allowing you to send follow-up emails, warranty reminders, or even targeted promotions without manual effort.

While this guide covers one example, you can customize it to choose which customers to send over and assign specific tags in Mailchimp for personalized email campaigns.

To complete this process, you will need:

  • RepairQ Professional

  • Zapier Premium

  • A Mailchimp account (Standard subscription required to automate email campaigns)

Create a new Zap in Zapier

A 'Zap' is an automated connection between two or more apps that allows them to work together without manual intervention.

Note: Currently, Code Steps in Code by Zapier are limited to returning 250 items in the output object. As a result, any customer list exceeding this limit will not run.

Start by logging into your Zapier account and creating a new Zap. Select Webhooks by Zapier as the app, then choose Catch Raw Hook as the trigger event. Click Continue to proceed, and Zapier will then create a Webhook URL. Copy it and set it aside.

Retrieve Customer Information in RepairQ Analytics

Log in to your RepairQ instance and navigate to the Analytics reporting section. Click the File icon in the upper right corner, then select Shared Folder. From there, click on the Customer Folder and choose the MC Customer look.

Note: You can use the Analytics section in RepairQ to create and schedule reports to send customer data for any campaign.

By default, this look pulls closed tickets from the previous month. To proceed, click the Gear Icon and select either Send or Schedule.

Select the data to go as a Webhook and paste the URL you saved from Zapier into the field that appears.  Then select JSON - Simple as the data format. Don't click send yet, since we’ll need to test in Zapier first

Test & Review in Zapier

Jump back to Zapier and click Continue, then click Test & Review.  Once you click it, head back to RepairQ and click Send.  Zapier should see the Webhook from RepairQ and load some test data. After confirming, click Continue.

Select Code by Zapier as the next app, and choose Run Javascript as the Action Event. In the following section, input rawData into the 'Input Data' field. Then, from the dropdown next to it, select Raw Body.

In the Code section, input the following:

const elements = JSON.parse(inputData.rawData);
const attDat = JSON.parse(elements.attachment.data);
return attDat.map(item => {
 return item;
});

In the next section, click Test and Continue, then add a third section with the plus button. Select Mailchimp as the app and Add/Update Subscriber as the Action Event.

In the next section link your Mailchimp account with Zapier.

Next, select which Audience you want your RepairQ contacts to go to. Then select Customer.email from the drop-down. Scroll down and click Refresh Fields.

New fields will appear. Select Customer.first Name and Customer.last Name for their respective sections.

Add a fourth Zap and select Mailchimp as the App. Use Add a Subscriber to Tag as the Action Event.

Use the same account you linked earlier in the next section.

In the next section, select the same Audience you used above. Input the Tag you want to attribute to customers from RepairQ. Then use Customer.email for the Email Address.

Click Next and Save the Zap.  Then use the toggle at the bottom to activate the Zap. Customers sent over will appear in the Audience you selected showing the source as Zapier and the tag you selected.
