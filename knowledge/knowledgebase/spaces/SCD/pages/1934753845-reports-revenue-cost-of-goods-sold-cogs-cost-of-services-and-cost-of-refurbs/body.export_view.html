We recommend accounting for revenue and COGS reports based on closed tickets only; once a ticket is closed the date will no longer change, and the sales are reported as revenue. By default, filters are set to closed tickets only, because it is the most reliable method for this business model. For further information about accrual-based reporting, see our article, RepairQ Accounting Principles.

Refurbish Ticket Behavior

In your Location Settings, you can control how RepairQ should track COGS for refurbished items. This allows you to transfer the cost associated with parts and labor consumed in the refurb process to the refurbished item's cost. This means that after the refurbishment, the cost saved for the refurb item could equal the purchase price for the refurb item, plus the cost of parts, labor, or otherwise. When the refurb item is sold, it will reflect your cost(s) invested in the item in the Cost of Goods Sold Report. If you see a zero cost reported for an item, then the item either 1) had no cost, or 2) the cost was transferred to the refurb item, which will show up on the COGS Report when the item is sold. It will not show up as a separate line item; it will be part of the refurb item's total cost.

Navigate to Location Settings → Location Info to modify Refurbish Ticket Behavior.

Viewing Revenue and COGS Reporting

1. On the Navigation Bar, click Reports.

2. On the Reports page, click on the Accounting dropdown along the top row of report categories. From the available list, there are four reports designed to assist with tracking revenue and COGS reporting: Revenue, Cost of Goods, Cost of Services, and Cost of Refurbs.

3. (Optional) When viewing a report, click Hide/Show Filters on the upper-right side of the page to collapse/expand the filters when viewing reports.

4. (Optional) When viewing a report, click To Excel on the upper-right side of the report page to export an Excel-friendly document. The report will include any filtered data when running the report.

Revenue Report

The Revenue Report allows you to look at sold tickets over a period of time. Use the available filters to scope results based on the assigned parameters. Once you have set your filters, click Apply Filters to run the report.

  • Date Range - Select a date range for your scope of results.

  • Date Type - Use in combination to find items within the date range.

    • Ticket Created - Used to sort tickets by the date it was created.

    • Date Updated - Used to sort tickets by the date it was last updated.

  • Location - Select a specific location or run an aggregate report for multiple stores.

  • Status - Select whether your data pulls from tickets in Closed status, Open status, or all statuses.

    • By default, this will pull data from closed tickets, since ticket information is finalized.

  • Billing Agent - Select any/all applicable Billing Agents.

  • Item Type - Search for a specific Item Type or multiple types.

The Revenue report shows the Total Net Sales, with a breakdown of totals for Gross Sales/Returns, Restock Fees, Net Discounts, and Net Sales. Each line of the report includes additional information about the item sold, as well as a link to view the original ticket.

Note: You can hover over each value to view a Tooltip with additional information and an equation for the amount.

Cost of Goods Sold Report

The Cost of Goods Report allows you to look at all non-labor items sold during a time period. Non-labor items include all items where the assigned Item Type is NOT flagged as Labor/Service. For more information on Item Types, check out our article, Catalog Settings Overview.

  • Date - Select a date range for your scope of results.

  • Location - Select a specific location or run an aggregate report for multiple stores.

  • Status - Select whether your data pulls from tickets in Closed status, Open status, or all statuses.

    • By default, this will pull data from closed tickets, since ticket information is finalized.

  • Billing Agent - Select any/all applicable Billing Agents.

  • Item Type - Search for a specific Item Type or multiple types.

The Cost of Goods Sold Report shows the Total Cost of sold items, with a breakdown of totals for Inventory Costs and Cost Adjustments. Each line of the report includes additional information about the item sold, as well as a link to view the original ticket and Inventory Summary page for the item. The Total Cost column shows the amount you paid for the item; items in red are returns.

Cost of Services Report

The Cost of Services Detail Report allows you to look at all Labor items sold during a time period. These include all items where the assigned Item Type IS flagged as Labor/Service. For more information on Item Types, check out our article, Catalog Settings Overview.

  • Date - Select a date range for your scope of results.

  • Location - Select a specific location or run an aggregate report for multiple stores.

  • Status - Select whether your data pulls from tickets in Closed status, Open status, or all statuses.

    • By default, this will pull data from closed tickets, since ticket information is finalized.

  • Billing Agent - Select any/all applicable Billing Agents.

  • Item Type - Search for a specific Item Type or multiple types.

  • Include Labor Costs? -  Select whether or not to include labor costs in reporting. Labor is added by hours entered on ticket items and the staff member's pay rate found in their Staff User Profile.

The Cost of Services Detail Report shows the Total Cost of labor items. Each line of the report includes additional information about the item sold, as well as a link to view the original ticket and Inventory Summary page for the item. The Add'l Cost column (additional fixed cost) is set by the Catalog Item cost. The Labor Cost column (when selected as included in the report) includes labor hours. Labor is logged in the ticket item quantity field and is multiplied by the hourly pay rate of the employee selected in the Repaired By section of the ticket item.

Cost of Refurbs Detail Report

The Cost of Refurbs Detail Report shows the cost of Ticket Items (parts, labor, accessories, etc.) added to Refurbish Tickets. This report helps you catch any costs that are not reported in the Cost of Goods Sold or Cost of Services Reports.

  • Date - Select a date range for your scope of results.

  • Location - Select a specific location or run an aggregate report for multiple stores.

  • Status - Select whether your data pulls from tickets in Closed status, Open status, or all statuses.

    • By default, this will pull data from closed tickets, since ticket information is finalized.

  • Item Type - Search for a specific Item Type or multiple types.

  • Include Labor Costs? -  Select whether or not to include labor costs in reporting. Labor is added by hours entered on ticket items and the staff member's pay rate found in their Staff User Profile.

The Cost of Refurbs Detail Report shows a breakdown of the total costs of ticket items added to Refurb Tickets. Each line of the report includes additional information about the item used, as well as a link to view the original ticket and Inventory Summary page for the item.

Note: The cost for parts/labor can be configured in the Location Settings so that it is transferred to the item refurbished when added to the Refurb Ticket. When this happens, the item on this report will display as zero cost because the cost was transferred to the item that was refurbished. When the refurbished item is sold, the cost added to the refurbishment will be reported in COGS. For more information, see Refurbish Ticket Behavior.
