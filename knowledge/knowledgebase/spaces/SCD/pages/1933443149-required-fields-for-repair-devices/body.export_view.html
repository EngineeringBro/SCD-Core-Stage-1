Note: This feature currently disrupts Qsite Opportunities features and will cause unexpected errors when creating an appointment. We are currently working to resolve this issue. If you are using Qsite, we recommend waiting to establish required fields until a fix has been released.

Admins can assign required fields to configure which device information is necessary for capture when adding a repair item to a ticket. A required field cannot be skipped when entering device information.

Repair device required fields are available to users on RepairQ Advanced and RepairQ Professional plans.

Updating Required Fields for Repair Devices

1. On the Navigation Bar, go to Settings → Company Settings.

2. On the Settings Overview page, the left column has additional categories. Click on Customers to expand the selection and click Device Required Fields.

3. On the Validation Rule screen, select any/all fields you wish to require when adding a repair device to a ticket. Once you have made your selection(s), Save your changes.

Note: By default, the Initial Diagnostics/Notes field for a repair device is always required.

4. When adding a repair device to a Repair Ticket, you will see required fields are noted with an asterisk (*). When selected, the text box for a required field will also have a red outline for easy reference.

Note: If the device is not working and you are unable to obtain certain details, the field will accept "DOA" (dead on arrival) or any other short description for the issue.

5. Required fields will also apply to existing repair devices in a customer's device history. These new fields will require information before you can continue with the repair.
