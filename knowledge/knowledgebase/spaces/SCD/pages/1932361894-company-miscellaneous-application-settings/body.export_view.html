Miscellaneous Application Settings is a section used to manage various search behaviors, ticket information, and staff permissions that do not fit into other categories.

  • All Miscellaneous settings are available at the Company level (System Admin access required).

  • Search behaviors are customizable at the Location level (Location Admin and Manager Access required).

RepairQ includes added security measures, allowing in-app authorization for Support Team members to access your database to assist with troubleshooting and support requests.

Accessing Company Miscellaneous Application Settings

Miscellaneous Application Settings are available for Admins and Location Admins by navigating to Company/Location Settings →  Miscellaneous.

Miscellaneous Application Settings

Manage filter search behaviors, Leaderboard Report settings, and viewing analytic information on tickets.

Search and Ticket Behavior Settings

For additional details on search behaviors for the below settings, check out our article, Keyword Search Techniques.

  • Combine words with two or fewer characters as a single keyword.

    • This feature is active by default. Unchecking the box will our Legacy Search functionality, which separates phrases with two or fewer characters as unique search terms. 

  • Show All Locations on the Leaderboard.

    • When enabled, this will allow store managers to see all locations on the Location Leaderboard Report.

  • Enable legacy customer search

    • This allows you to search by the current algorithm where multiple keywords and keyphrases are accepted when searching, or revert to our legacy algorithm where each word is split into separate search terms. 

  • Show Cost on VCT

    • This setting allows you to show/hide the Profitability Analytics section of tickets as well as the Cost section of the Price Check window when a VCT is assigned to the computer. For additional information, check out Hiding Analytics on Tickets and Price Checks.

Staff and Password Management Settings

Manage automatic termination of inactive Staff Profiles, switching profiles, and Password/PIN security requirements.

  • Mark Staff “Terminated/Not Employed” after N days of activity - Automatically deactivate staff after a set number of inactive days. Users can no longer access the environment if they show no System Log activity based on the Total Days assigned.

  • Enable Fast User Switching - Activate or deactivate the Switch User option from the Navigation Bar. When active, this allows fast switching between users if a user is already logged into the system.

  • Allow PIN-only user switch and timeclock - When selected, staff will be prompted to enter only their PIN/Password to switch users or when clocking in/out of the system. When unchecked, the user must enter their Username and PIN/Password.

  • Show optional Username field with PIN-only - When selected, this setting works with the above feature to show both the optional Username and required PIN/Password fields when switching users or clocking in/out. The user can enter both fields or leave the Username field blank and only enter their PIN/Password.

  • Require unique PIN - When activated, users must create a unique PIN/Password when establishing login credentials. When unchecked, users can create a PIN/Password that is also in use by another user.

    • If you leave this feature deactivated, we recommend disabling “Allow PIN-only user switch and timeclock” to avoid conflicts if a user enters an identical PIN/Password to another user.

  • Require strong PIN/Password - Enforce a stronger format for user passwords.

    • Passwords must include at least 6 characters, contain at least one uppercase letter and one lowercase letter, and must contain at least one symbol (~!@#$%^&*()-_=+[{]}\|.;:‘“,.<>/?).

RepairQ Support Login Settings

Manage RepairQ Support Team login access for troubleshooting.

  • Allow trusted Support Team members to log in separately (most secure & traceable)

    • This allows the Support Team to log in as an Admin for the database. The System Log will record which team member logged in and record any changes they make within your environment.

  • Allow impersonation by trusted Support Team members

    • This allows the Support Team to log in as a specific user, under their assigned role. This is valuable in troubleshooting issues that particular users or user roles are experiencing.
