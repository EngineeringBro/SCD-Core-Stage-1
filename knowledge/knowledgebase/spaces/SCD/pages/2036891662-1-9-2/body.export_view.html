Released 01/04/2016 4:15PM UTC 10:15 AM CT

This is a minor update to address bugs and add minor features.

Customer Sync on Tickets

In some cases, when creating a new customer on a repair ticket, the customer would not show as attached to the ticket until the repair device is added. This was due to a race condition in the customer sync process, and it has been resolved.

Deleting Ticket Notes

Fixed an issue with removing ticket notes on a new ticket (not yet saved for the first time).

Ticket Edit Lock Upgrades

RepairQ protects against saving conflicting edits of the same ticket, where two different users save the same ticket at the same time. This protection has been extended to the scenario where the same user saves the same ticket from two different windows.

Ticket Import Upgrades

Three new fields have been added to the ticket import feature:

  • Repair Cost Estimate

  • Repair Device Location (or Bin #)

  • Ticket Notes

Inventory Search + Bulk Edit

Fixed an issue where the bulk edit button would initiate the "print labels" action if you performed the following sequence:

  1. Use the global search to pull up an inventory item by SKU match (Where RepairQ places you on the item page rather than a search results page)

  2. Select items to bulk edit

  3. Click bulk edit button

Now when the above sequence is performed the bulk edit action will occur, rather than the print labels action.
