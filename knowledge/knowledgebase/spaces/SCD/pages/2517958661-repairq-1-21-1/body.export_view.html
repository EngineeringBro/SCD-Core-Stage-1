New Features

Discounts

A complete overhaul of our discount system has been implemented for Essential, Advanced, and Professional users. This new system is optional; if you prefer the previous method of applying discounts, you can continue manually adding discounts at your discretion for any item or service, within their maximum discount limit. When the new Discount feature is active, new discounts must be created for your staff to apply to products and services. Benefits of the new feature include:

  • Create Discount Codes with customized discount amounts, durations, and applicable products and services.

  • Staff can add discounts by searching the Discount Name or using the Discount Grid.

  • Discount Reporting shows which discounts were applied to items.

Staff Management

The Roles section of the Staff Management page has been redesigned.

  • The User Roles matrix has been replaced with a new search field. This field will function as a single-select lookup for assigning locations, allowing users to search available locations for role assignments instead of scrolling through a list of locations.

  • When assigning a location role, a new radio button is used to designate the Primary Location of the staff member. This clarifies a user’s primary location when viewing reports and analytics.

The Staff List now includes a new configurable column displaying the primary location assigned to staff members. Additionally, a filter has been added to enable users to search based on their primary location.

A new Company-level setting can automatically deactivate staff profiles after a set number of days of inactivity. (Settings → Miscellaneous → Miscellaneous)

Repair Tickets

New Company-level ticket setting, “Auto-erase passcode field for customer devices at Device Returned/Closed status” to automatically remove saved passcode entries for customer devices upon entering the “Device Returned” step (for claims) or closing a Repair Ticket. (Settings → Tickets → Ticket General Settings)

Enhancements and Changes

Administrative Roles

  • System Admin Access - This is the updated authorization for full administrative access within the system. Previously, this was named “Admin Access,”

  • Admin Access - This is an authorization for limited administrative access across all locations in the system. This includes the Manager role plus additional authorities:

    • Location Settings access for all locations, excluding settings for creating new Ticket Approvals, Ticket Actions, and Ticket Resolution Types.

    • Can create, read, update, and delete Ticket Notes and Approvals.

    • Has full inventory count permissions - Count, edit, reconcile, etc.

Customer Marketing and Communication Settings

Updates to customer communication preferences for separate notification approvals for Service Email/SMS and Marketing Email/SMS. Default preferences are managed in Location Information details.

Tickets

The customizable “Appointment Date” and “Appointment Time” columns are merged when viewing the Ticket Queue.

Bundle Required Ticket Status - A new setting in Ticket General Settings allows you to assign specific ticket statuses that restrict advancement until an item flagged as “Bundle Required” is added to the ticket. This ensures that tickets cannot advance until all required parts have been bundled.

Custom Forms

Custom Form Fields for Text Input, Text Area, and HTML now have a Dynamic Value dropdown, allowing users to insert shortcodes into the text area, allowing for more customized forms. Shortcodes will be replaced with the appropriate data depending on the context of the form.

Catalog Items

Added a “Year” attribute to Catalog Items. This enhancement will sort device models according to their year attribute (YYYY) when navigating Item Grids.

Notifications

Created a new Automated Notification shortcode {device_description} to pull the device description when creating a notification.

Admin Updates

Only Admin users can add and edit Billing Agents. This change is implemented to prevent reporting errors caused by unauthorized changes.

Location Updates

The location requirement when filtering the Staff List has been removed. For Admins, filtering with a blank location field will search all locations. For store-level users, the blank filter will search all locations where they have access.

Invoicing

Added an invoice import feature that will allow a user to create an invoice by importing a list of tickets that are in “Waiting for Payment” status for the same billing agent.

Purchase Orders

A new Global/Location Purchase Order setting (Settings → Catalog → Purchase Orders) allows users to set the item Condition when creating a PO, before placing the order. The default setting allows users to assign the Condition for items when receiving items on the PO.

Bug Fixes

Lists and Reporting

For users without Admin access, the Staff Overview page correctly reflects the number of filtered locations and members for users under the Locations and Member count.

Square tip amounts and processing fees are now accurately reflected on the Journal Entry Report.

Ticket Items and Notifications

When a serialized inventory item is attached to a ticket and then removed, re-adding the same serialized item and saving the ticket incorrectly shows the item status as "Instock. The item status will now correctly be saved as "Pulled.”

Users can now successfully set Additional Triggers for Notification Automation, ensuring notifications are sent as expected when multiple conditions are met.
