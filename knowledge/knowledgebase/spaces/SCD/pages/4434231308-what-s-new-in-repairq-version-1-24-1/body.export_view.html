Released April 2026

TL;DR

This release focuses on reliability and control. We cleared out a wide range of issues that were interrupting everyday work — imports that stalled, tickets that wouldn't open, integrations that stopped connecting — and shipped new tools for managing how your team works: label printing, form behavior on conversions, read-only field controls, purchasing controls, and payment integration fixes. French language support has been significantly expanded, and Global Payments reliability improvements are included for all operators.

New Features

Custom Form Enhancements

Custom forms can now be configured to appear after a ticket is created, rather than at the exact moment of saving. Fields that automatically pull in ticket details (device type, customer name, repair category) now populate correctly instead of coming up blank. If your team has been filling those in by hand because they weren't auto-populating, this is the fix you've been waiting for.

Note previews directly in the ticket list

Tickets now show a note preview right in the list view, so your team can see the latest note and how many notes are on a ticket without having to open it. Click to expand. Useful for busy queues where context needs to travel with the ticket.

In-app guidance is back

The contextual tooltips and guided walkthroughs throughout the app are re-enabled. If your team leans on these for onboarding or for brushing up on less-used workflows, they're available again.

Purchase orders now prevent over-receiving

When receiving inventory against a purchase order, RepairQ now enforces the ordered quantity limit. Staff can no longer receive more items than were ordered — even if multiple browser tabs are open at the same time. This closes a gap that was allowing accidental over-receipt of parts and devices.

Full audit trail for inventory counts

Every inventory count now has a complete history log showing when it was created, who ran it, and when it was cancelled if applicable. This gives managers full traceability for stock discrepancies and audit reviews.

French language support significantly expanded

RepairQ's French interface now covers significantly more of the platform. App Settings, Reports, System Logs, Notification Queue, and other key areas are now available in French for operators who have French enabled. If you support French-speaking staff or customers, far less of the interface will fall back to English.

Improvements

Working with tickets

  • Opening a ticket for editing no longer produces an error. All tickets can be opened and edited normally.

  • Creating a new ticket from a customer profile works again, even when that customer already has open tickets.

  • Converting a quote to a repair ticket now works regardless of which location you are signed into. Previously, you had to be logged into the same location the quote was created at.

  • Adding items to a ticket no longer throws a JavaScript error. All item types can be added without issue.

  • Completing a Quick Sale no longer returns a server error.

  • A template bug that was causing incorrect refund amounts when bundle rules were active is fixed. Refunds now reflect the correct figures.

  • Removing a payment plan from a ticket now works correctly. The error that was preventing removal is resolved.

Admin and setup controls

  • Admins can now mark specific location fields as read-only for non-admin users. This prevents store-level staff from editing fields that should only be managed by administrators.

  • A new setting lets you disable quote deletion entirely. When enabled, quotes must be converted to a ticket or voided — they cannot be deleted. This keeps a complete record of all quoted work.

Ticket labels and printing

  • Four repair lifecycle dates — Device Received, Service Started, Repair Completed, and Device Picked Up — can now be included on printed ticket labels. Useful for shops that print labels at intake or handoff.

Custom forms

  • Custom forms and any collected signatures now carry over when a repair ticket is converted to a claim. Previously these were lost on conversion.

  • Image and photo upload fields in custom forms now display correctly when a form is sent for external review.

Importing customers and inventory

  • Customer imports now accept all valid data. Fields like Country, Referral Source, and Referral Customer were rejecting correct entries or producing confusing errors. These are resolved.

  • Import jobs no longer get stuck on "Scheduled" indefinitely. Jobs that previously stalled without progress or error feedback now run through to completion.

Payments and integrations

  • Square payment notifications are now reliably saved to tickets. Events that were silently dropped are now recorded correctly every time.

  • Connecting or reconnecting QuickBooks Online works again. Stores that were hitting an error during authentication can now complete the setup without issue.

  • Backordered items now show their correct price. Items that were displaying $0.00 now show the actual price at checkout.

  • A warning now appears if you try to disable the virtual terminal while an integrated payment processor is still active. This prevents an easy-to-miss misconfiguration that would break payment processing.

  • Global Payments debit refunds no longer fail with a missing transaction reference error. Refunds that were being approved by the processor but failing to save now complete correctly end to end.

  • The instructions displayed when canceling a Global Payments transaction now reference the correct button in the interface.

  • Your location's language setting now correctly applies to customer-facing receipts and invoices.

Inventory and purchasing

  • The recovery reasons button works again. Selecting a recovery reason no longer produces an error.

  • When completing a return and marking the device as In Stock, the associated refurbishment ticket now closes properly instead of staying open.

  • "Needs Attention" tickets that have been overridden by an admin no longer revert back after moving to the "Waiting for Payment" status. The admin override is preserved.

  • Parts that are in stock are no longer incorrectly flagged as Backordered when completing a repair.

  • The Price Check "In Stock only" filter now works correctly when you have multiple locations selected.

Customers and billing

  • The customer detail page no longer returns a server error on load.

  • The invoices page loads correctly without a 500 error.

  • Discounts load correctly when editing or creating tickets. The error that was preventing the discount list from appearing is resolved.

Claims and service workflows

  • ServicePower claims are routing to RepairQ again. A credential update had caused some locations to stop receiving claims. Credentials now save and apply correctly.

  • Likewize claim creation is working again for service locations that were unable to create claims.

Appointments

  • Staff with appropriate access can now bypass the minimum advance notice requirement when creating or editing appointments on behalf of customers.

Notifications and performance

  • The terminal assignment prompt no longer appears on devices where the virtual terminal feature is turned off.

  • A behind-the-scenes fix reduces a pattern of repeated duplicate requests on certain pages. Some users may notice a performance improvement as a result.

  • System labels throughout the interface now display correctly. Fields that were showing internal key names instead of readable text are resolved.

French language display

  • French interface text now displays correctly in Reports, System Logs, and customer-facing label views. Several pages that were showing broken or missing French text are fixed.

  • App Settings pages for Payments, Tickets, Taxes, Categories, Catalog, and Languages sections now display correctly in French.

  • Errors that were appearing on the customer screen tickets page and in App Settings when French was enabled are resolved.

Questions or issues?

If you are on version 1.24.1 and still seeing any of the issues above, reach out to RepairQ Support and we will take a look.
