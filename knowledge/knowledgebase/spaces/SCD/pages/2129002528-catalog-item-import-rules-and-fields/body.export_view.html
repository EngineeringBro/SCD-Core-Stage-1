RepairQ offers convenient inventory management by allowing users to add and update Catalog Items via an import file. Users can effortlessly create catalog items, registering the items and services in your store. This process is streamlined through import files, supporting .xlsx (Excel), .csv (comma-separated values), and .txt (tab-separated values), with .xlsx recommended for the best experience.

  • Efficient Inventory Management - Easily add, modify, and maintain items in your Catalog.

  • Streamlined Data Entry - Quickly upload items in bulk, reducing errors and saving time.

  • Flexibility - Keep up with the latest devices and services as they become available.

Import capabilities are restricted to users with Admin, Manager, and Inventory Manager roles. This ensures that only authorized personnel handle critical data operations, maintaining data integrity and security.

Catalog Import Template

For instructions on downloading the import template, see Catalog Management: Importing Catalog Items.

When working with the Catalog Import Template, you must have the correct header names (provided in the template) to import the column values successfully. The order of the columns does not matter as long as the correct information is under the assigned header name. Only the values you wish to add or update need to be present in the import file, and any unrecognized column headers will be ignored.

Catalog Import Rules

When importing data, RepairQ will compare the information to existing entries first. If no matches are found for a record in your import file, RepairQ will create a new catalog item record from the import row.

  • Column Matching - Columns are matched by header name. We recommend using the header names provided in our templates to ensure a successful import of the column values. Matches are made when comparing the following categories, in this order: SKU, Supplier SKU, and Name.

  • Required Values - Required values for creating a catalog item are Item Type, Manufacturer, Name, and Active.

  • Unrecognized Columns - Any columns added outside the provided template will be ignored.

  • Import Updates - If a line in the import file matches an existing record in RepairQ, the item will be updated with the data from your spreadsheet. Matches are made in the following order: SKU, Supplier SKU, and Name.

    • When updating an existing entry, one of these fields must be present and match the corresponding Catalog Item you wish to change. For example, if you are trying to update an Item Name, including the matching SKU will help identify the item, then an update will be made to the name, rather than a new item created.

  • Accepted File Formats - RepairQ accepts Excel (.xlsx), Comma-Separated Values (.csv), and Tab-Separated Values (.txt). We recommend using .xlsx files for the best upload experience.

Required Import Template Fields

The following fields are required to create a new catalog item in RepairQ:

  • Item Type - This is a classification of the catalog item. This must match an existing Item Type in your Catalog settings.

  • Manufacturer - Assign the manufacturer of this catalog item. This must match an existing Manufacturer in your Catalog Settings.

  • Name - This is the primary name to identify the Catalog Item. This must be unique for each item.

  • Active - Active Catalog Items are searchable in lists and can be added to tickets.

    • Use the number one “1” to mark the item as active.

    • Use the number zero “0” to mark the item as inactive. This is used when retiring an item from use. An inactive catalog item cannot be added to a ticket and will be removed from the default Catalog List view.

    • Deactivating a Catalog Item does not remove the inventory of that item. If retiring an item, we recommend purging the inventory first.

    • You can still locate and reactivate items by filtering your Catalog list for Attibrute = Inactive.

Optional Catalog Item Fields

These additional fields aid in comprehensive product management. As RepairQ evolves, this list will change:

  • UPC - Universal Product Code. This unique identifier is used to track stock and inventory of your items.

  • SKU - Stock Keeping Unit. Similar to a UPC, this unique identifier offers a more detailed identification than barcodes. SKUs can include product details like size, color, and style, providing additional inventory info.

  • Supplier SKUs - This is used for integrated Purchase Orders. Catalog items can be linked with a Supplier SKU to identify the same item from your supplier’s inventory when placing an order with a supplier.

    • Data in the Supplier SKU field must include the Supplier name and SKU in the following format: “SupplierNameSKU”. e.g., “Apple:AP123”.

    • You can add multiple Supplier SKUs, separated by commas: “SupplierName:SKU,SupplierName:SKU”.

  • Friendly Name - This is an alternative, customer-friendly name shown on receipts.

  • Categories - This is used for establishing an Item Tree. Categories organize your Catalog Items in parent/child relationships. Item Trees are the underlying structure of navigating Item Grids in RepairQ.

    • Your Item Tree must be established before assigning categories to items since the data must be spelled exactly as it is in RepairQ.

    • Data in the Categories field must be in the following format: Category>Subcategory>Subcategory>etc. e.g., Smartphones>Apple>iPhone 8

  • Search Keys - Additional searchable keywords for the Catalog Item. This can be used when searching your inventory page, for an item to add to a ticket, etc.

    • Multiple search keys must be separated with a comma, without spaces. e.g., 'searchkey1,searchkey2,searchkey3'.

  • Repairable - Marking a Catalog Item as repairable allows users to add it to a Repair Ticket.

    • Use the number one “1” to mark the item as repairable. Use the number zero “0” to restrict the item from being added to a Repair Ticket.

  • Inventoried - Items marked as inventoried have a quantity that can be tracked in RepairQ. These items must be stocked to be used on tickets and can be replenished via Purchase Orders, inventory adjustments, and Inventory Counts. Non-inventoried items can be added to tickets however, there is no way to track the amount of this item in your possession.

    • Use the number one “1” to mark an item as inventoried. Once this item is added to a ticket or purchase order, this field will be locked for editing to maintain data integrity.

    • Use the number zero “0” to mark an item as non-inventoried. This is typically used for labor, services, and other items that do not require a physical quantity on a ticket. Once this item is added to a ticket, this field will be locked for editing.

  • Serialized - Items marked as serialized will require a serial number when the item is added to your inventory and added to a ticket.

    • Use the number one “1” to mark an item as serialized. Only items marked as inventoried can be marked as serialized. Once this item has been added to a ticket or purchase order, this field will be locked for editing.

    • Use the number zero “0” to mark the item as non-serialized. Once this item has been added to a ticket or purchase order, this field will be locked for editing.

  • Reorder Point - (For Inventoried items) Reorder Points are designed to notify managers on the dashboard and during the PO process when the in-stock quantity for an item falls below the assigned reorder point quantity.

  • Reorder Amount - (For Inventoried items) Reorder Amounts are designed to make ordering easier by assigning a default quantity to reorder when adding an item to a Purchase Order.

  • Price - This is the retail price of the item. This is the amount that the customer pays to purchase the item. This may be affected by factors like profit margin, market demand, etc.

    • If a price is entered for the Catalog Item, it will automatically be applied when adding the item to a ticket or into inventory.

  • Advertised Price - By default, the “Price” field is used for standard retail pricing. If the item has a special, advertised price on Qsite, that price can be entered here.

  • Is Advertised Price - This flags the item to use the Advertised Price when shown on Qsite.

    • Use the number one “1” to allow for advertisement on Qsite. Use the number zero “0” to restrict the item from being advertised on Qsite.

  • Price Including Tax - If utilizing Tax-Inclusive Pricing, you can enter the after-tax price of the item here. RepairQ will refactor the price and tax of the item based on your assigned taxes for the Item Type.

  • Cost - This is the expense of your business to produce, acquire, or deliver that item. This may be affected by factors like your purchase price of the item, materials, labor, overhead, and other associated costs.

    • If a cost is entered for the Catalog Item, it will automatically be applied when adding an item to your inventory.

  • Restocking Fee - Restocking fees are often used when a return requires significant effort to bring the product back to a sellable condition. Assigning a restocking fee to a Catalog Item will charge that additional amount when the item is returned on a ticket.

  • Default Qty/Hours - This allows you to assign a default quantity/labor hour amount when adding the item to a ticket.

  • Allow Cost Override - This allows you to change the cost of the item when added to a ticket.

    • Use the number one “1” to allow for Cost Override. Use the number zero “0” to restrict the item from overrides.

  • Allow Supplier Override - This allows you to change the item's supplier when added to a ticket.

    • Use the number one “1” to allow for Supplier Override. Use the number zero “0” to restrict the item from overrides.

  • Note - If you have an additional note for this Catalog Item, you can add it in this field.

  • Bin - This is generally used for inventory organization. If you plan to keep this item in a designated area (bin), you can label that location with this field.
