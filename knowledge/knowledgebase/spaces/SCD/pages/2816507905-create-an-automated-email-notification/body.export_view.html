Users can send email notifications directly from a ticket to inform customers about their ticket status, notify them when their device is ready for pickup, or send a follow-up message thanking them for their business. To send an email notification, the following must be met:

  • A Customer Profile must be linked to the ticket.

  • The option ‘Receive Service Email Notifications’ must be selected in the Customer Profile.

  • The Customer Profile must contain a valid email address.

    • Please use valid customer email addresses when creating or editing profiles to avoid bounces, spam reports, and potential throttling of your message sending. Using fake or placeholder emails like ‘na@na.com’ can lead to delivery issues by flagging our emails as spam.

Automated Notifications are intended for transactional and relationship messaging (ticket updates, pickup notifications, internal alerts), not for marketing or promotional content.

For details and regulatory context, see Getting Started with Automated Notifications

Set up an Email Notification

From the Navigation Bar, go to Settings → Company Settings. On the Settings Overview page, under the Communications section, click Notifications.

On the Notification Automation Overview page, both stock and user-created notifications are listed. You can either create a new notification or edit an existing email notification. To create a new email notification, click Create Notification Automation.

Automated Notification Parameters

Start by filling out the notification fields. You'll need to set the Trigger and Delivery Method, which are required. Other fields are optional and can be used to set more specific conditions for sending the notification. Additionally, provide the message details for the outgoing communication.

  • Notification Name - Choose a name that reflects the purpose of the notification.

  • Location - Select whether the notification applies to a single location or all locations. Admins can create notifications for all locations, while managers are limited to locations they have access to.

  • Trigger - Choose the action that will trigger the notification. You can select from various options, including Ticket Status Change, Inventory Status Change, Appointment Notifications, and more.

    • In the example, we chose Ticket Status Change to trigger the notification when a ticket is saved in a specific status.

  • Ticket Actions - Select the specific ticket status that will trigger the notification.

    • In the example, we chose Ready for Pickup to send an email when the ticket reaches this status.

  • Additional Trigger - Select the ticket type that will apply to this notification (Sale, Repair, Trade-in, All, etc.)

  • Warranty Providers - If the notification is related to a repair or service that involves a warranty provider, select the applicable warranty provider here.

  • Ticket Resolution - If you assign a resolution to a ticket (e.g., Completed, Cancelled, etc.), select the resolution that will trigger the notification.

  • Delivery - Select the delivery method for the notification.

    • In this example, we chose email based on the above selections.

Message Details

Begin by filling out the Message Subject, Send To, and Email Message fields. Customize these fields based on the notification you'd like to send.

Note: Each section provides a list of available Short Codes that you can copy and paste into the message. These short codes are placeholders that will be replaced with the corresponding data when the notification is sent. Be sure to include the brackets when copying/pasting the short codes into your message for proper functionality.

  • Message Subject - This field defines the subject or title of the notification. Ensure that it reflects the purpose of the message, as this will appear in the customer’s notification.

  • Send To - Use the available short codes to specify the destination for this notification. Shortcodes can include customer information, specific user roles within the software, or the location email address. Ensure that the correct shortcode is used to direct the message appropriately.

  • Attachment - You can attach a PDF, such as a receipt, invoice, or checklist, if needed.

    • In this scenario, no attachment is necessary, so leave this field blank unless required.

  • Email Message - Enter the content of the email notification here. You can personalize the message by using short codes to automatically insert customer details or ticket information. Make sure the message is clear, concise, and relevant to the notification being sent.

Send Delay

If you wish to send follow-up notifications, you can set a delay by specifying a timeframe in Days, Hours, or Minutes. Leave this field blank if you want the message to be queued and sent immediately.

Ensure the Active checkbox is selected, then click Save to create your notification.

Notification Queue

To check your outgoing notification queue use the Navigation Bar to go to Reports → Notifications.

On the Notifications Summary page, you can view a queue of all outgoing notifications. By default, new notifications remain in New Status for a few minutes before transitioning to Sent Status. This allows you a brief window to cancel a notification if you decide it should not be sent.

Note: You can cancel a queued message by clicking the Red X Button next to the notification.
