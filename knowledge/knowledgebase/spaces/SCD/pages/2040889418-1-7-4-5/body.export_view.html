This update contains a few bug fixes.

Reports

  • Default Filters - On all reports where there is a Ticket Status multi-select filter (e.g. most sales reports), the default (before you apply any filter changes) filter setting shows only the "Closed" ticket status selected. However, by default these reports would filter for all statuses excluding the "Void" ticket status. This has been fixed by properly applying only the "Closed" ticket status by default.

  • Invoice Aging Report - Fixed some issues with grouping invoice values, which would sometimes inflate the amount presented for a billing agent.

Billing Agents

  • The figures presented for ticket count and ticket amount on the billing agent index and on the billing agent view were sometimes incorrect. This has been fixed.

Enterprise Invoicing

  • The "write off" function was not working properly for tax exempt tickets. When you clicked the "write off" button it would calculate tax at the normal rate rather than the ticket tax rate override.

  • The totals presented at the bottom of the ticket list on the remittance screen would show for all tickets on the invoice, rather than showing for the tickets in the current view. This has been updated so that you can view totals for the current list of tickets in the list view.
