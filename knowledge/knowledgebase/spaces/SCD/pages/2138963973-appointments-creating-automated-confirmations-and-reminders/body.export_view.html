Creating appointments in RepairQ offers convenience and flexibility for customers by allowing them to reserve time with your team at their convenience. The ability to automate notifications reduces the time it takes to send communication, which allows for efficient communication between teams and improves the customer experience. Creating automated appointment confirmations and reminders helps your business in many ways, including:

  • Streamlining your operations by sending automated reminders for appointments.

  • Reducing the likelihood of no-shows for appointments

  • Enhancing the customer experience through timely communication and reminders.

This article provides a streamlined process for creating appointment notifications and reminders. For an in-depth walkthrough of creating Automated Notifications, see our article, Automated Notifications: Creating an Automated Notification.

Automated Appointment Confirmation

Navigate to Location Settings → Communications → Notifications. On the Notification Automation Overview page, click Create Notification Automation.

Notification Name and Behaviors

These fields determine the triggers and delivery methods for your notifications.

  • Notification Name - Assign a name for this notification that notes that it’s used for appointment confirmations.

  • Location - Select which location(s) this notification will be assigned to. By default, you can select any location you have a role. Admins can select a specific location or all locations.

  • Trigger - Select the trigger that will initiate this notification. In this case, we will trigger the notification with an Appointment Status Change.

  • Ticket Actions - Select the status that will initiate this notification. In this case, we will select Open appointment status, since this is the default appointment status for new appointments. Based on your configuration, this may be a different status. Admins can customize this in Appointment General Settings.

  • Delivery - Select the delivery method for your notification.

    • Email - Deliver the notification via email. The customer’s profile must have a valid email address for successful delivery.

    • SMS - Deliver the notification via SMS. The customer’s profile must have a valid SMS phone number, SMS carrier, and approval to receive SMS messages for successful delivery.

    • Email + SMS - This will send an outgoing notification to both delivery methods based on the information in the customer’s profile.

Notification Subject and Delivery Details

These fields accept shortcodes to automatically populate contextual data from RepairQ into your notification. Accepted codes are listed above each field.

  • Message Subject - Enter the subject of your outgoing notification.

  • Send To - This is the outgoing recipient of the message. Using the shortcode {customer} will pull contact information from the customer’s profile when making the appointment.

  • Reply To - This determines the recipient of any replies to the original message. Using {location_email} will send any replies to the email address assigned in your Location Information details.

Notification Message and Send Schedule

These fields accept shortcodes to automatically populate contextual data from RepairQ into your notification. Accepted codes are listed above each field. You can also customize the timing of outgoing notifications.

  • Email Message - Create the body of your outgoing email for the customer. You can use short codes like {customer_first_name}, {location_short_name}, and {scheduled_date} to pull information about the appointment and insert them into the email.

  • SMS Message - Create the body of your outgoing SMS message for the customer. You can use short codes like {customer_first_name}, {location_short_name}, and {scheduled_date} to pull information about the appointment and insert them into the message.

  • Send Delay - By default, outgoing notifications are queued immediately. If you would like to delay your outgoing messages, you can select your delay time in minutes, hours, or days.

  • Active - If this field is checked, the automation will run as intended. If you need to stop this notification, you can uncheck this field to turn off the automation.

Automated Appointment Reminder

Navigate to Location Settings → Communications → Notifications. On the Notification Automation Overview page, click Create Notification Automation.

Notification Name and Behaviors

These fields determine the triggers and delivery methods for your notifications.

  • Notification Name - Assign a name for this notification that notes that it’s used for appointment reminders.

  • Location - Select which location(s) this notification will be assigned to. By default, you can select any location that you have a role for. Admins can select a specific location or all locations.

  • Trigger - Select the trigger that will initiate this notification. In this case, we will trigger the notification by an Appointment Notification.

  • Ticket Actions - Select Appointment Reminder for the action.

  • Delivery - Select the delivery method for your notification.

    • Email - Deliver the notification via email. The customer’s profile must have a valid email address for successful delivery.

    • SMS - Deliver the notification via SMS. The customer’s profile must have a valid SMS phone number, SMS carrier, and approval to receive SMS messages for successful delivery.

    • Email + SMS - This will send an outgoing notification to both delivery methods based on the information in the customer’s profile.

Notification Subject and Delivery Details

These fields accept short codes to automatically populate contextual data from RepairQ into your notification. Accepted codes are listed above each field.

  • Message Subject - Enter the subject of your outgoing notification.

  • Send To - This is the outgoing recipient of the message. Using the short code {customer} will pull contact information from the customer’s profile when sending the appointment reminder.

  • Reply To - This determines the recipient of any replies to the original message. Using {location_email} will send any replies to the email address assigned in your Location Information details.

Notification Message and Send Schedule

These fields accept short codes to automatically populate contextual data from RepairQ into your notification. Accepted codes are listed above each field. You can also customize the timing of outgoing notifications.

  • Email Message - Create the body of your outgoing email for the customer. You can use short codes like {customer_first_name}, {location_short_name}, and {scheduled_date} to pull information about the appointment and insert them into the email.

  • Reminder - Set your reminder to send in minutes, hours, or days before the scheduled appointment.

  • Active - If this field is checked, the automation will run as intended. If you need to stop this notification, you can uncheck this field to turn off the automation.

The Notification Queue

You can check outbound notifications by navigating to Reports → Notifications. The Notification Queue shows all outgoing communications, including notification type, message preview, attachment, and status. Notifications in New Status can be canceled by clicking the red cancel button next to the notification. Once the notification has been sent, it cannot be undone.
