Efficient inventory management is crucial for businesses of all sizes, and RepairQ offers a range of methods to facilitate moving inventory to a central location. Each method is designed to cater to different scales of operation—whether you're dealing with small batches or larger transfers.

Benefits of Centralizing Inventory

  • Centralized inventory helps ensure that stock levels are consistently accurate across all locations, reducing discrepancies and errors.

  • By consolidating inventory, businesses can reduce carrying costs, minimize excess stock, and optimize reordering processes.

  • Streamlined workflows for receiving, transferring, and managing inventory can save time and reduce labor costs.

Manual Attribute Changes

For quick adjustments to inventory, the Inventory Detail screen allows managers to update the attributes of individual items or perform bulk changes across multiple records of the same inventory item. This feature is useful for updating Inventory Status, Condition, Location, Supplier, Price, Cost, and Carrier. All changes are automatically logged in the Change Log, providing a record for transparency and audit purposes.

Key Benefits:

  • Time-Saving: Update multiple records in bulk, significantly reducing the time spent making individual adjustments.

  • Enhanced Control: Manage various inventory attributes in one place, ensuring your inventory is accurately represented.

  • Consistency: Apply uniform changes across similar items, helping to maintain a structured and consistent inventory.

Use Case

This method works best for small-scale movements since it requires working with each Catalog Item, individually. It’s an efficient way to update the location for large quantities of items, however, there’s no way to verify if the items were physically moved, as RepairQ reflects the change without validation at the receiving. It lacks a check-and-balance system, which may lead to discrepancies between the system and actual inventory.

Change Inventory Attributes via Import

For larger bulk updates and inventory movements, we recommend utilizing an Import Template. This process allows you to make bulk changes to inventory attributes, including Inventory Status, Condition, Location, Supplier, Price, and more. This method involves downloading, updating, and re-importing data. Although the initial process may take time to update the template, it allows you to make changes across multiple Catalog Items, rather than working with a single item at a time.

Import Rules

  • Import Template: Download the provided template, update the necessary fields (such as item location), and re-upload the file to process the changes.

  • Inventory Status Rules: You can update the attributes of items in Live or Transition status. Items marked as Consumed (Sold, Voided, Shrinkage, etc.) cannot be modified to maintain data integrity.

  • File Formats: Accepted formats include Excel (.xlsx), Comma-Separated Values (.csv), and Tab-Separated Values (.txt). For the best experience, we recommend using the .xlsx format.

Use Case

This method is ideal for larger inventory transfers, where multiple items need to be updated at once. It’s useful when dealing with large volumes of inventory or when changes to attributes such as location need to be applied in bulk. However, similar to manual updates, this process does not verify if the items have physically been moved to the new location. The system reflects the changes, but there is no built-in confirmation process for the physical transfer.

Inventory Transfer (Internal Purchase Order)

The Inventory Transfer process is the most thorough method for transferring inventory between locations, ensuring accurate and verified inventory movements. Though time-consuming, it provides more control and accuracy by allowing stores to request specific quantities of items based on real-time stock availability at the supplying location. This method includes an approval process, shipment tracking, and reconciliation to ensure both sending and receiving locations maintain accurate inventory levels.

How It Works

  • Internal Transfer Request: Users on our Advanced or Professional plan with multiple locations can request inventory from another store or designated warehouse. The requesting store specifies the quantity and type of items needed, and the supplying location must approve or reject the request.

  • Approval Process: The supplying store reviews the request, and they can:

    • Approve the request and ship the requested items.

    • Partially fulfill the request by shipping available items and rejecting the rest.

    • Reject the request if unable to supply any of the items.

  • Shipping Costs: The supplying store can notify the receiving store of shipping costs, and the receiving store can log on to the internal purchase order (PO) before reconciling the transfer.

  • Receiving Inventory: Once the items arrive, the receiving store checks the shipment to confirm accurate inventory levels before reconciling the transfer.

Use Case

This method is best suited for multi-location businesses needing precise control over inventory transfers. For example, if Store A needs to restock from Store B, the system ensures that the transfer is verified, approved, and tracked at every step. Unlike simpler methods, this process allows both parties to review and confirm the transfer, ensuring transparency and accuracy in inventory counts and shipping costs. This process provides the most thorough verification, but it also requires more time and coordination between locations.
