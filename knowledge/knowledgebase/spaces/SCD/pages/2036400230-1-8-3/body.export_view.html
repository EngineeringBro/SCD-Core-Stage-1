Scheduled Release: 8/29/16

Introducing, the new Global Search UI

Our goal is to provide quick access to any working document in the system from any page in RepairQ using a global search, and to get there we need to make a few changes. In RepairQ 1.8 we added several new time saving features such as the new "Jump To" functionality, where if only one record is found RepairQ places you on the resulting record instead of a search results page, reducing several clicks and page loads along the way. We also made an "advanced search" accessible so that any complex search you would normally perform from a list page could instead be performed from anywhere. RepairQ 1.8 also improved the global ticket search so that you can search for tickets by ticket number, customer, and device serial, instead of just the ticket number. However, we recognize that along with these improvements, we took a step backwards in a few areas related to searching records efficiently, and this update addresses those areas.

We have two primary goals with Update 1.8.3

Reduce the number of clicks before searching

Before 1.8, you were one click away from an inventory or customer search, where you could simply click the appropriate search field to begin typing. With the new Global Search in 1.8, it would require three clicks before searching: one click to open the dropdown, one click to select your search target, and one click to focus on the search input before typing. In this 1.8.3 update, we reduce the number of clicks back down to one. First, you click your search target, and then you can begin typing because the search input is automatically selected for you.

Reduce accidental search submissions

In 1.8, the search target label doubled as the submit button for the search. This design tended to get in the way of efficiency. Before this update, it was placed next to the search target selection, and being the larger clickable area made it too easy to click it on accident when attempting to change the search target selection. In the 1.8.3 update, we added a smaller dedicated submit button, and we moved the search target selection to a new toggle button away from the submit button.

Multi Select Filter Update: select/unselect all

To facilitate the global search updates in 1.8, we replaced the multi-select checkbox inputs with new search driven multi-select inputs, but in the process we removed the select all and select none options. In RepairQ 1.8.3, all of the new search driven multi-select inputs have a select all and select none option.
