RepairQ’s import feature simplifies customer profile management by enabling users to batch-add records, streamlining data entry and reducing the need for manual input. This functionality is especially beneficial for businesses transitioning from other systems, or for those needing to upload large volumes of customer data at once. It ensures that customer information, including contact details, service history, and billing data, is efficiently transferred to your POS system, saving time and reducing errors compared to manual data entry.

Users with Admin, Manager, or Inventory Manager roles have access to this feature, allowing them to import various types of records, including Tickets, Customers, Purchase Orders, and Inventory. Our built-in import templates are designed to ensure that data is formatted correctly for seamless upload into RepairQ. This feature is a key tool for businesses looking to keep their customer database up-to-date, especially when onboarding new clients or consolidating data from multiple sources.

Import Guidelines

Columns are matched on the header name, so you must have the correct header names (provided in the template) to import the column values successfully. Any unrecognized column headers will be ignored; only the values you wish to add or update need to be present in the import file. If no matches are found for a record in your import file, RepairQ will create a new Customer Profile.

Download the Customer Import Template

From the Navigation bar, click Customers. When viewing the Customer List, click the Options button ( ⋮ ) on the top right of the page to expand the list of available options. From the list, click Import.

A new section will appear below your Customer List. Click Download Import Template download an Excel file with pre-defined column names. This file is formatted specifically for entering ticket data, ensuring it aligns with our system's requirements.

Customer Import Template

After downloading the blank Customer template, open the file and add items to add to the document. The following columns require data to create a new Customer Profile in RepairQ: First Name, Last Name, and Primary Phone. After adding the data, be sure to Save your changes.

Required and Suggested Fields

The template file will include the columns available for import. Required values for creating a catalog item are First Name, Last name and Primary Phone.

  • First Name (Required) - The customer's first name, used to identify the individual in your environment and when linked to tickets.

  • Last Name (Required) - The customer's last name, used to identify the individual in your environment and when linked to tickets.

  • Primary Phone (Required) - The main contact number for the customer. This number is used for direct calls regarding repair updates or service inquiries.

  • Email - The customer’s email address, which can be used for sending service updates, receipts, and marketing materials.

  • SMS Phone - The phone number designated for SMS communications, including service notifications, reminders, and promotional messages.

  • SMS Carrier - Required for the SMS notification feature. This field identifies the carrier (e.g., AT&T, Verizon) associated with the customer's SMS phone number to ensure messages are delivered correctly.

  • Any additional fields you have marked as "Required" in the Customer Required Fields settings.

Import the Customer Template

When you have saved the file and are ready to upload your import, click Choose File and select your file from the file explorer. Once the file is selected, click Import. RepairQ will begin processing your import file. You will be notified via email when the import is complete, and you can check the Import Queue for any additional details.

Update Existing Customers

In addition to adding new customer profiles, the import feature can also be used to update existing customer records in RepairQ. This is especially useful when bulk-editing customer information, such as updating contact details, loyalty card numbers, or other profile attributes.

When you upload an import file, RepairQ checks for matches between the data in your file and the existing customer records in your database. Matches are determined based on the following criteria, in this order:

  • Customer ID - The unique identifier assigned to each customer profile. This is the most reliable way to match records during the import process.

  • Loyalty Card - The loyalty card number associated with the customer. If the Customer ID is not provided but a matching loyalty card number is found, the system will update the corresponding record.

  • Primary Phone + Last Name - If neither the Customer ID nor the Loyalty Card number matches, the system will check for a combination of the customer’s Primary Phone and Last Name to identify the correct record.

Matched Records

If a match is found using one of the above criteria, the system will update the existing customer profile with the new or changed information provided in your spreadsheet.

Unmatched Records

If no match is found, a new Customer Profile will be created in the system using the data in the import file.
