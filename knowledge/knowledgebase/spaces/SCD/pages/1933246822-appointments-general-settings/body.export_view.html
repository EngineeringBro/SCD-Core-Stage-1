Users can reserve blocks of time for your customers to come in and spend time with an assigned user. You can also specify whether an appointment is for a Repair ticket, Sales ticket, or a Trade-in ticket. If a customer decides they need to change their reservation, users can change the appointment from the Appointment List.

Appointment General Settings allow you to change the parameters around scheduling appointments for customers. You can customize appointment status defaults, duration of your appointment windows, grace periods, and more.

Appointment General Settings

From the Navigation Bar, go to Settings → Location Settings.

Note: A user with an Admin role can change these settings at the Global level by going to Company Settings instead.

On the Settings Overview page, look for the Appointments section. Under this area, click General Settings.

Appointment General Settings

Default Appointment Status

Default Appointment Statuses allow you to select the default status when creating, initiating, and canceling an appointment.

  • Default Appointment Status (New) - This is the default status when creating a new appointment.

  • Default Appointment Status (Converted) - This is the default status when an appointment is taken from the queue and helped by an employee.

  • Default Appointment Status (Canceled) - This is the default status when an appointment is canceled.

Appointment Duration

Appointment Duration settings allow you to create windows of time for scheduled appointments, assign a default duration when creating an appointment, and limit the number of concurrent appointments.

  • Duration Options - This designates the amount of time assigned for appointment intervals. For example, 15min, 30min, or 1hr. Set the duration and click +Add to create the duration time. You can create multiple duration windows.

  • Default Appointment Duration - Default amount of time given to an appointment. More options will be available as you add more duration options.

  • Concurrent appts - This is the number of appointment windows allowed to happen at the same time. Setting a limit to concurrent appointments restricts scheduling to avoid overlapping appointments. For example, one concurrent appointment allows for one appointment at a time.

No-Show Grace Period

  • Calculation Method - Select whether the grace period will apply towards the time of the appointment or the window.

  • No-Show Grace Period - Select the amount of time the appointment can still be started after the customer has missed their time/window.

Appointment Calendar View

These settings allow you to customize what information is visible when viewing the appointment calendar and what action the user can take when clicking an appointment.

  • Appointment Calendar View - The following checkboxes, when checked, will allow information about the appointment to show up within the calendar blocks. If multiple fields are selected, this information may only be visible when viewing the Week and Month view.

    • Customer Name - The customer the appointment is reserved for.

    • Assignee Name - The staff member the appointment has been assigned to.

    • Customer Service Address - The reserved customer’s address in the Customer Profile.

    • Device Name - This is the repair device assigned when creating a Repair Ticket Appointment.

  • When clicking on an appointment - Select which action is available to the user when clicking on a scheduled appointment.

    • Show Summary - This will show the Appointment Summary popup with details for the customer.

    • Show Edit Screen - This will show you the Appointment Edit popup when clicking an appointment.

  • Show the option to start the appointment - Toggle the option to start the appointment when selecting an appointment from the calendar view.

Note: For more about scheduling and viewing appointments within the calendar, please see Scheduling Appointments.

Appointment Advanced Notice

  • Appointment Advanced Notice - Create guidelines for when an appointment can be created using the following parameters in minutes, hours, or days.

    • Store Open "buffer" - This prevents appointments from being opened in the buffer period after opening the store.

    • Store Close "buffer" -This prevents appointments from being opened in the buffer period before closing the store.

    • Minimum Advanced Notice - Appointments cannot be scheduled at a time before the minimum notice. For instance, if the minimum notice is 1 hour, the appointment cannot be scheduled within one hour at the time of creation.

    • Maximum Advanced Notice - Appointments cannot be scheduled after a time exceeding the maximum notice. For instance, if the maximum notice is 14 days, the appointment cannot be scheduled after 14 days from the time of creation.
