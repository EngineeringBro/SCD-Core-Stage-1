Managing staff profiles in RepairQ is key to maintaining an organized, efficient workflow and ensuring your team operates seamlessly. By creating Staff Profiles, you can control user access, track productivity, and manage labor costs effectively.

By creating individual profiles, managers can utilize the following key benefits:

  • Assign Roles and Permissions - Assign roles to staff members for single or multiple locations, granting them specific actions in RepairQ.

  • Assign Pay Rates - Assigning pay rates to users helps calculate labor costs accurately.

  • Clock In/Out with Timesheets - Users can clock in and out of RepairQ using their timesheet. This feature allows managers to track hours worked and maintain an accurate Profitability Report.

  • Track User Actions - Managers can monitor user actions in the System Log, ensuring they know when tasks are completed and who is responsible.

  • Track Productivity - Tickets can be attributed to staff members, enabling tracking of sales and repairs.

Create a new Staff Member (Profile)

On the Navigation Bar, go to Staff → Add Staff. Alternatively, when viewing the Staff Overview list, click +Add New Staff on the right side of the page.

New Staff Required Fields

Other fields in the details area allow you to add contact information for notifications that can be sent via SMS to the User.

  • First Name/Last Name - The name of the staff member.

  • Email - This field is used for Password resets and Notifications.

  • Username - This is the assigned username for the staff member, which will be used when logging into RepairQ.

  • PIN/Password - This can be a PIN code or an alphanumeric password, based on your password settings. Staff members should not share passwords.

  • Confirm PIN/Password - Enter the same password to confirm consistency.

Employee Information

Enter the relevant details for the user, including their assigned roles (e.g., Manager or Shift Lead), pay rate, maximum discount allowance, and whether the staff member can record worked hours when using RepairQ.

  • Status - Select whether the user is Active or Inactive. When inactive, the user cannot log into the system.

  • Assigned Manager - Select the manager for this staff member. This will assist with reporting.

  • Hourly Pay Rate ($) - Add the user’s hourly pay rate. This assists with profitability reporting for labor.

  • Max Discount - Enter the maximum discount percentage or dollar amount available for the user when discounting items.

  • Allow Price Overrides - This option allows users to override ticket item pricing. This action replaces the current price with a new price and is different than applying a discount.

  • Timesheet Required - Select this field to allow the user to clock in/out to record worked hours.

Assign Location-Based Roles

Assigning specific roles to users determines their responsibilities and permissions within the software, ensuring they have access to the necessary tools and information to perform their tasks efficiently. This helps maintain security, streamline workflows, and clarify job expectations. For more information on the capabilities of each role, see Roles and Permissions.

Note: Non-Admin users must have at least one role assigned at a location to log in.

Use the Location Dropdown to assign a location for the staff member and click +Add. The location will then be added to the user's list of location assignments. In the Role field, assign a role (or multiple roles) to the user for each applicable location. Use the Primary radio button to designate the Primary Location for the user, which will be used to identify their primary location in reports.

All roles have default access to the following operations:

  • Basic Operation - Log hours (if the timecard is active), perform price checks, and update their User Profile photo.

  • Customers - View, create, and update customers, customer notes, and customer devices.

  • Inventory - View the inventory summary, perform inventory counts, and add inventory to tickets.

  • Tickets - View and search the Ticket Queue, view, create, and update ticket information, create and update ticket notes, add/remove ticket items, process a ticket sale/return/return repair, print/email/SMS a receipt.

Admin Access

Special access permissions allow overrides of the current staff roles for more granular control. Using this feature, Admins can assign standard roles to users while granting temporary special access permissions to override their default role.

  • System Admin Access - The user will receive override access to all areas of RepairQ for all locations.

  • Admin Access - The user will receive Location Settings access for all locations, excluding settings for Ticket Approval, Ticket Action, and Ticket Resolution Types.

  • Accountant Access - The user will have view-only access to tickets and reporting for all locations.

  • No Special Access - The user will receive no additional permissions outside their assigned roles.

IP Whitelist

This is an approved list of IP addresses to manage where your Staff Member can access RepairQ. This adds security to your database by granting users access from specific terminals.

Note: If you do not know the address of your current terminal, hover over the Tooltip on the IP Address column to view your current IP and Wildcard IP address settings. Admins are not required to have an assigned IP Address.

Under IP Whitelist click on Add New IP to add a Connection Name and approved IP Address. You can add additional addresses for multiple locations and terminals.

Deactivate a Staff Member

Deactivating a User will restrict their access to RepairQ and increase your available User Count. We recommend deactivation rather than removing all roles and IP Addresses since this does not properly update your User Count and may cause issues when updating your RepairQ Subscription.

Note: Staff members can be automatically deactivated after a set number of inactive days, depending on your Login Settings.

When editing Employee Information, use the Status dropdown to select Terminated/Not Employed and Save your changes. The user will be restricted from logging into the system and is hidden from staff dropdown lists. You can reactivate a user by searching your Staff List for inactive users and editing the user profile.
