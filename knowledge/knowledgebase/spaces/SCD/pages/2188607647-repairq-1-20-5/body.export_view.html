Our commitment to providing you with the best possible experience continues with upgrading our software to PHP 8.1. This update brings a range of improvements and enhancements to make your experience even better!

  • Enhanced Performance - Faster loading times and smoother interactions. Enjoy a snappier and more responsive software experience.

  • Advanced Security - Stronger protection for your data, creating peace of mind for you and your customers.

  • Future-Proofing - Ensuring our software remains up-to-date with industry standards opens the doors for better integrations and new features.

With this update, we’re also introducing our new RepairQ Starter plan, designed for those new to the repair industry. Repair and Quick Sale tickets allow you to track standard business operations, manage potential business via quotes and in-app appointments, and view basic reporting for tracking sales and other metrics. Additional features can be added as your business grows and your services and capabilities expand. For more information, check out our article, 2024 RepairQ Subscription Plans.

Features

  • Clone a Custom Form - Save time and effort by creating copies of forms while retaining established form details and fields.

  • Clone a Catalog Item - Copy an existing Catalog Item, saving time and reducing repetitive data entry. Cloning items ensures consistency in your catalog by carrying key information to new items, reducing the risk of errors.

  • Reopen Closed Tickets and Reconciled Purchase Orders - Admins can update Ticket and PO settings to allow managers to reopen closed records. This feature puts more control in the hands of our users, allowing for greater flexibility in managing operations without contacting the Support Team.

Enhancements

  • Various improvements to Qpop for performance, stability, and connectivity.

  • Improved the way RepairQ processes inventory data to create inventory counts. This will prevent high-volume inventory counts from impacting performance.

  • Improved the consistency of Location filter results. Previously, databases with a higher number of locations would omit some locations from search results

  • Added “Reject All” when selecting line items in an Internal Transfer (internal Purchase Order). This improves the efficiency of rejecting items on an internal purchase order.

  • Catalog items now allow for multiple Supplier SKU assignments.

Bugs

Catalog, Inventory Items, and Purchase Orders

  • Unless specified, the Catalog Item List will no longer include deleted items in filtered results.

  • Merging catalog items has been fixed.

  • Deleting catalog items will no longer result in an Error 500 message. If the user receives an error, we have also updated the error handling process to give the user an explanation of why the item could not be deleted.

  • Fixed the unresponsive “Add Stock” button when manually adjusting inventory item quantities.

  • Fixed an issue that allowed a user to remove the “Supplier” field details when editing an inventory item.

  • Activating auto-serialization will no longer restrict the user from manually assigning serial numbers when adding/editing serial numbers for inventory items.

  • Purchase Orders have been fixed to restrict users from adding new items after progressing to Ordered status.

  • Rejecting an item on an Internal Purchase Order will no longer zero out the “Quantity to ship” fields of the page.

Tickets and Invoices

  • Customer notes flagged as important will no longer show HTML tags when linked to a ticket.

  • Fixed the “waive fee” button on tickets.

  • Fixed a bug that prevented the user from creating an invoice after selecting and then deselecting all items from the ticket list.

  • If Auto-serialization is turned off, the user will no longer see the incorrect message that a serial number will be automatically generated when recovering a part core on a ticket.

  • Fixed a bug that ignored the option to add prefixes to serial numbers when using the auto-generate feature.

Appointments and Customers

  • Concurrent appointment fields have been renamed “Maximum Appts” for clarity and consistent behavior across all available settings.

  • Other Various fixes to the Appointment Scheduling process.

  • Fixed a bug where zip code validation interfered with customer validation rules that did not require a zip code to create a customer profile.

Custom Forms

  • Resolved the issue causing broken images upon uploading a photo to a custom form.

  • Resolved the issue causing a blank “Text Area” field data on custom forms.

Lists and Filters

  • Fixed an issue in List Filters that caused stray text when typing inside of multi-select search boxes.

  • Corrected the column titles in the inventory import template for successful uploads.

  • Sorting the Item Type List by the “Bundle (as Parent)” column no longer results in an “Error 500.”

  • Resolved an issue that caused categories to show as blank when editing columns on various List View pages.

  • Firefox users will no longer see blank “Viewing” and “View” dropdown options when filtering lists.

Integrations and Automation

  • QZ print automation has been fixed, and we’ve included additional error handling to help identify any future problems that could arise with the automation.

  • Twilio can now be managed at the Location level, rather than the integration only working at the Company level.

  • Resolved an issue that logged duplicate transactions in the Cash Reconciliation Report if there were duplicate, active integrated payment methods.

  • Fixed a bug that failed to remove tickets in Critical status from the Ticket Progress widget on the dashboard.

  • Resolved an issue that restricted Automated Notifications from sending if they were queued near midnight UTC.
