A recent update to Chrome Browser may interfere with the transaction process when using an integrated PAX device in RepairQ. The PAX device may try to request payment numerous times, resulting in duplicate/orphaned transactions, or RepairQ may prompt you with an error that the card must be removed. There are settings within Chrome browser that you can disable to reduce the likelihood of these issues.

When Chrome sends a Private Network Access preflight, the PAX device treats the preflight like a regular request, which doubles the requests. The PAX device will allow for multiple requests to queue, and it will present any queued requests after the previous request finishes. This can result in double charges if the customer swipes twice, successes that appear to fail initially (the first swipe is successful; the second one fails), and also orphaned transactions where the first is successful, but RepairQ looks at the status of the second, failed transaction.

Updating Chrome Browser Settings

1. On the Chrome address bar, type chrome://flags and press Enter on your keyboard.

2. On the Experiments page, use the Search Bar at the top of the page to search for the keyword preflight.

3. From the list of three (3) results, change all of the settings to Disabled.

4. At the bottom of the page, click the Relaunch button to save your changes and launch Chrome with these changes in effect.

5. If you experience any additional issues, we recommend using an alternative browser (Firefox, Safari, or Edge Browser).
