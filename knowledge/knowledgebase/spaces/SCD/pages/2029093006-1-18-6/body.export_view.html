Released 05/23/2022 at 11:30pm CT

Improvements

  • Improved Square transaction handling to reduce “Error 500.”

  • Improved Square transaction process to prevent a user error possibility, which ended in duplicate transactions.

  • Improved Square transaction process to prevent "failed transactions" from resolving multiple times with client handling, which ended in multiple unique transactions.

  • The Cash Reconciliation Report now uses proper report dates for its Gateway transactions.

Bug Fixes

  • Fixed an issue where the Reconciliation Report reflects Square transactions inaccurately.

  • Resolved a bug where transactions would sometimes duplicate with the same Transaction ID.

  • Square no longer gets stuck in a confirmation loop, and no longer attempts to charge the customer again when stuck in this loop.

  • Improved the Switch User feature so that the Assigned VCT does not switch with the employee.

  • Updating Billing Terms for customers now saves the updated information properly.

  • The Import Customer Template now works again as intended.
