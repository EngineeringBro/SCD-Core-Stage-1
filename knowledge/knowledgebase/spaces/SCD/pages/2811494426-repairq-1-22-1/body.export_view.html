This release addresses key fixes across several areas, including the Virtual Cash Terminal (VCT), accepting partial cash payments, receiving Purchase Orders, and Import/Export functionality.

Cash Terminals and Payments

  • Implemented an automatic Workstation Key reassignment to maintain active VCT sessions. This update prevents issues where the VCT may unintentionally close or un-assign itself, allowing users to continue their work without requiring a log out and log back in.

  • Resolved an issue where users encountered an erroneous "Float amount is set to $0.00" error, which prevented VCT opening or reconciliation. This update also fixes an issue where the cash reconciliation report was missing, ensuring a smoother VCT process and accurate reporting.

  • We fixed an issue where adding a partial cash payment incorrectly set the ticket balance to $0. Now, only the entered payment amount will be applied, ensuring the remaining balance is correctly reflected.

Purchase Orders

  • We resolved an issue where some purchase order items were not receiving a timestamp when marked as received, which caused errors for users processing purchase orders.

Imports and Exports

  • We resolved an issue where queued import and export tasks showed no records after completion. Previously, exports generated a success email, but the link directed users to a broken page, and imports consistently failed. Both issues have now been addressed for smoother task processing.
