Adding Integration Credentials in RepairQ

Note: Admins have the option to add Account ID and API Key information at the Company level, which will apply these settings all locations on your database. However, the Group ID must be filled out at the Location level since it is unique to each location and should not be shared.

This tutorial will walk you through adding all of the required integration credentials at the Location level, to ensure that each location will be set up with correct details.

1. Log into the RepairQ location that will be using this payment integration. On the Navigation Bar, to go Settings → Location Settings.

2. On the Settings Overview page, click Integrations under the Account section.

3. Under the page of Available Integrations click Bluefin.

4. Enter your integration credentials based on the information provided to you by Bluefin Support and/or RepairQ Support. Your Account ID may be individual or shared, based on the structure created by Bluefin’s Implementation Team. If you have multiple locations, each location will have a unique Group ID which must be entered at the Location level for the integration to be successful. Once you are finished with your selections, click Save Bluefin Settings to submit your changes.

  • Account ID - This is the primary numeric ID for your account and any secondary locations (sub-accounts) under the same gateway. This number is provided to you by Bluefin Support and is accessible in your payment gateway. It may also be provided to you from RepairQ support, as a courtesy.

  • API Key - This key is used to authenticate the integration between RepairQ and your Bluefin gateway. You will use the API Key provided at the Account level.

  • Group ID - This number is unique to each location in the Bluefin payment gateway. In most cases, the primary location will have the same Account ID and Group ID. Sub-accounts will share the same Account ID but will have a unique Group ID for their location.

  • Allow Tipping - This will allow/disable the tip function on Bluefin transactions. If you enable tipping for the first time, you will also need to contact Bluefin Support for a software update for your device.

    • By default, when tipping is active, the following tipping logic is used:

      • If the transaction is less than $10, the customer is presented with tip options for $1, $2, $4, and a custom dollar amount.

      • If the transaction is greater than $10, the customer is presented with options for tip percentages of 5%, 10%, 15%, and a custom dollar amount.

  • Tip Refund Logic - This allows users to determine how the tip will be split and recorded in RepairQ when performing a refund on a ticket.

    • Refund Tip Amount First - Refund the tip amount first, then the refund on the sale amount.

    • Refund Tip Amount Last - Refund the sale amount first, then the tip amount last.

    • Refund Tip Amount Pro Rata - Refund the tip in proportion to the sale amount refunded to the customer. e.g. A customer makes a $10 purchase and gives a 10% tip of $1.00. A partial refund of $6.00 is performed on the purchase. Square will refund a part of the tip that is proportional to the remainder of the purchase. 60% of the original amount was refunded ($6.00), so 60% of the tip will also be refunded ($0.60).

  • Prompt for debit or credit - If this setting is inactive, all card transactions will be processed as Credit. If this setting is active, the customer has the choice whether to process their payment as Debit or Credit.

    • Prompt on RepairQ - This setting will add an extra prompt in RepairQ for the casher to ask the customer whether they would like their payment processed as Debit or Credit. There is also an option that can push that prompt to the payment device for the customer to select.

    • Prompt on Payment Device - This setting will add an extra propmt on your payment device for the customer to select whether their payment will be processed as Debit or Credit.

Note: Depending on your account structure set up by Bluefin, multiple locations may share the same Account ID. The Group ID, however, should be unique to each location and should not be shared across multiple locations.

5. Repeat this step at each location you wish to enable. Once you have added the integration information, you are ready to download and install the PAX Security Certificate.
