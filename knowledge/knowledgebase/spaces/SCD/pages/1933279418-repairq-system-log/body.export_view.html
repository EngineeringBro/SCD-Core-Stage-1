The System Log is a comprehensive record of changes and activities within RepairQ, capturing events such as Staff login details, Ticket modifications, Customer Profile updates, Catalog adjustments, Inventory changes, and more. The System Log also includes filtering options, allowing users to find specific actions within a defined date range. Additionally, it provides direct links to affected tickets, inventory items, and customer profiles, making it easier to access and review modifications. The System Log is accessible to users with Admin, Manager, and Shift Lead permissions, ensuring that key personnel can effectively monitor and manage system activities. Benefits include:

  • Accountability - Track changes and activities, hold users accountable for their actions, providing a detailed audit trail for compliance and review purposes.

  • Security - Monitor suspicious activities, enhancing security and preventing unauthorized access.

  • Data Integrity - Maintain data integrity by tracking all changes made to the system

Viewing the System Log

On the RepairQ Navigation Bar, go to Reports → System Log.

System Log Data

At a glance, the log shows details regarding changes made including the date of the change, who made the change, and the previous and current value of the attribute that was changed. You can use the page navigation links at the bottom of the log to jump to a specific page or scroll through consecutive pages.

System Log Filters

On the left-hand side of the page, filters will help narrow down the list of data you can view. Narrow your results by selecting filters and click Apply Filters to refresh the page with results that fall within those parameters.

  • Location - Select to scope log changes for all (any) locations or select a specific location that you have access to.

  • Staff - Select to scope changes made by a specific staff user or all (any) users.

  • From Date/To Date - Select a start date/end date for the list of results. These can be set independently to filter information from a certain date to the current or from the past up until a certain date. Leave these fields blank to search for all time.

  • Item - Select a category related to updates logged in the Changed Item column. This will look for specific events, like user log-in, purchase order changes, updating catalog items, etc.
