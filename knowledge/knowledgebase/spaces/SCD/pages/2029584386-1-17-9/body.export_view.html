1.17.9 is a minor update including some optimizations based on user feedback since release 1.17. The areas of focus for this update are list filtering and search, as well as modal windows.

List Views

  • Lists have new Quick Actions.

    • Lists that have more than one action, like view and edit, now have a new setting allowing you to select action that will show in your list’s ‘Actions’ column. Like editing columns, Quick Actions are a way to personalize your lists and can be saved with a view.

  • List filters are now more keyboard friendly.

    • You can tab between filter “tags”

    • Pressing the DELETE or BACKSPACE key on your keyboard while focused on a tag will remove the filter

    • Pressing the RETURN key while focused on a tag will open the user interface to adjust the filter (except for keyword tags)

  • List keywords are now more keyboard friendly, too.

    • Pressing RETURN will add and apply the keyword(s) and load the results

    • Pressing TAB will add the keyword(s) and allow you start adding a new keyword

    • Pressing ESC will clear any text you entered and dismiss the keyword input

  • Resetting a filter now removes pending filters.

    • The Reset button is now available when adding new pending filters. Clicking it will remove all pending filters and refresh your list.

Modal Windows

  • Modals are now faster.

    • We removed a ½ second animation when showing and hiding modals (screens that pop over RepairQ to capture specific information. Modals are now displayed immediately.

    • When a modal is opened, like Device Details, your cursor now starts on the most important field on the screen hopefully eliminating an extra click. Examples of cursor focus include: Required inputs, non-required inputs, Submit/Continue button.

    • While in the Device Detail modal, after typing your diagnostic notes, you can now press Return on your keyboard to Add Device. This along with changing the starting position of the cursor should make adding a device to a ticket faster.
