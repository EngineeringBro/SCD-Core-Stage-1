Released 01/31/2017 11:30 CT (02/01/2017 5:30 AM UTC)

This is a minor update to address bugs and add minor features.

Integrated Printing UI Updates

Based on your feedback we have a few user interface updates for integrated printing from the ticket view. Our goals with this update were:

  • One click to print a receipt/invoice/label to a default printer

  • The option to print to a different printer in the event your default printer isn't working (e.g. offline, out of paper, etc.)

The new interface goes back to (prior to integrated printing) separate print buttons for each print template, with a new attached menu for selecting alternate printers. If you click on the button with the icon (print, invoice, label) then it will print directly to your default printer for that print template. The default printer is defined by the RepairQ print configuration you have assigned to your computer. If you have no print configuration assigned, then the print button will default to the browser's print dialog.

Inventory Count Sheet

Several upgrades to inventory counts are here!

Inventory Zero Counts

The system used to ignore any items with a zero count, but now you can submit items with a zero count for reconciliation.

Load Count Assignments

We added a new feature for loading assignments into a count sheet. There is a new "load assignments" button added to your count sheets that will load up all count assignments for the current staff member for the current location into the count sheet. Each item will be loaded with a zero count, and as you scan items (or key in the quantity by hand) the quantity will increment normally. 

Now if you are assigned to count an item and none are available to count, the count sheet will include the item for you as a zero count, and managers have everything they need to reconcile inventory counts.

New Inventory Count Filters

Two new inventory count filters are available:

Date Range Filter

Inventory counts can now be filtered by a date range, and by one of the following date types: due date, counted date, reconciled date

Variance Filter

Now you can filter by the variance of the count; filter for counts that were short/zero/long

Purchase Order - Receive All

We have removed serialized items from the "receive all" feature because you must enter a serial number for each item received.

Ticket Analytics Update

We are excluding damaged item costs from the "Net Profit" estimate on the ticket view screen. This change is being made so that the ticket analytics uses the same calculation for cost as is used in the ticket profitability report. We now display damages as a separate line item in the analytics section, and it is not included in the "Net Profit" calculation.

Our original goal when including damages in "Net Profit" was to provide you with a figure that included any items damaged by a tech during the repair, but before we can achieve that on tickets and reports in the same way, we need to introduce some additional tracking to separate damages from defectives. Before we decide on the strategy for accomplishing this, we want to learn more about how everyone tracks defective items on tickets, and we will be sending out a survey to gather this feedback.

Our goal is to provide you with a clear picture of items damaged by a tech during a repair vs. items damaged elsewhere vs. defective items returned for RMA, and we will be working toward this in a future update. 

Dashboard - Purchase Order Alert

We now limit the number of pending (not closed/reconciled) purchase orders displayed on the dashboard.
