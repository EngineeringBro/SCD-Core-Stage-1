Ticket General Settings allow you to configure the behavior and appearance of Quick Sale, Trade-in, and Repair Tickets. While you can customize these fields based on your preferences, businesses in countries with GST/VAT must ensure that tax-inclusive pricing is displayed with field names that align with your pricing structure and comply with GST/VAT guidelines. This provides clarity for both your staff and customers.

In the examples below, we'll focus on GST-friendly labeling. You can independently customize labels in both Ticket View and Ticket Edit modes. However, we recommend using the same labels in both views to prevent confusion.

Note: We suggest applying these settings at the Company level to standardize labeling across all locations. However, individual locations can still customize their fields as needed. These changes are purely visual and will not impact pricing calculations.

Customize Ticket Item Fields and Labels

For these examples, we'll focus on GST-friendly labeling. However, you can customize the names of these fields according to your preferences. Please note that these changes are purely visual and will not affect the pricing calculations.

Log into RepairQ as an Admin. On the Navigation Bar, go to Settings → Company Settings. On the Settings Overview page, click General Settings under the Tickets section.

Customize Ticket Edit Mode

On the Ticket General Settings screen, locate the 'Customize Ticket Edit' section. Here, you can adjust the wording of various fields specifically for Ticket Edit Mode. The following fields are customizable and will update according to your selections.

Note: If any fields are left blank, they will use the default name for the column, which is visible in the text box. These examples use a tax-inclusive pricing structure for GST.

Display Layout

  • Default View - The available selections will change ticket column fields and labels based on your pricing structure.

    • Default - This will show unit price (without tax), Quantity/Hours, Tax, Discount, and Line Total.

    • Tax-inclusive - (Recommended for GST) This will show tax-inclusive unit pricing, effective after-tax discounts, etc.

    • Custom - This will allow you to toggle on/off which fields are visible on the ticket and customize the labeling of each.

Ticket View Columns & Label Overrides

  • Tax-Inclusive Unit Price - This is the unit price, before the discount.

    • For GST pricing, we recommend using ‘Unit Price (Inc GST).’

  • Quantity or Hours - This is the quantity of the item added or labor hours applied to a service/labor. Leave this field blank to use the default label.

  • Effective After-tax Discount - This is the discount ($ or % amount) applied to the item/service, before or after tax is applied based on your selection.

    • For GST pricing, we recommend using ‘Discount (Inc GST).’

  • Tax - This is the tax applied to the item’s retail price.

    • For GST pricing, we recommend using ‘GST.’

  • Line Total - This is the line total for the item after all applied discounts and taxes.

    • For GST pricing, we recommend using ‘Line Total (Inc GST).’

Ticket View Totals

  • Tax-inclusive Ticket total Without Discounts - This includes the price of all items on the ticket before discounts are applied.

    • For GST pricing, we recommend using ‘Subtotal (Inc GST).’

  • Total Effective After-tax Discount - This is the total of any discounts applied toward ticket items.

    • For GST pricing, we recommend using ‘Discount (Inc GST).’

  • Total Tax - This is the total for all taxes applied to the ticket items.

    • For GST Pricing, we recommend using ‘GST.’

Other Tax-Inclusive Settings (Ticket Edit Mode)

  • Show default Ticket Discount dropdown options, instead of below options, when tax-inclusive pricing is not enabled for the ticket’s location - This allows you to show/hide the default dropdown options, instead of the below two options, if tax-inclusive pricing is not enabled for your location.

  • “Before Tax” Ticket Discount dropdown option - This is used when applying a discount at the ticket level for all items. This will apply a discount before taxes are applied. This option has a toggle to show/hide this selection on the ticket edit screen.

    • For GST, we recommend using ‘Before GST.’

  • “After Tax” Ticket Discount dropdown option - This is used when applying a discount at the ticket level for all items. This will apply a discount after taxes are applied. This option has a toggle to show/hide this selection on the ticket edit screen.

    • For GST, we recommend using ‘After GST.’

  • Label for “After Tax” (Tax-Inclusive) Unit Price checkbox when editing Unit Price, if tax-inclusive pricing is enabled - This is used when overriding the unit price for a single item. This affects the Unit Price Editor field if you have variable pricing enabled for the catalog item (Can Override Price). If this option is checked when editing the unit price, the amount you enter will adjust for price + tax.

    • For GST, we recommend using ‘After GST.’

  • Label for Effective “After Tax” (Tax-Inclusive) Discount when editing item Discount, if tax-inclusive pricing is enabled - This is used when applying a discount for a single item. This affects the Discount Editor field when adding a discount to an item, the discount will adjust for the amount + tax.

    • For GST, we recommend using ‘After GST.’

Customize Ticket View Mode

You can customize labels independently in Ticket View and Ticket Edit mode. However, we recommend using the same labels in both views to avoid confusion.

  • Use the same settings for Ticket View as selected for Ticket Edit above - Choose whether to use the same settings for Ticket View mode as those selected for Ticket Edit Mode, or customize these labels separately.

    • When this is toggled ON, all of the Ticket Edit customizations will stay consistent on the Ticket View screen and the Ticket Edit screen.

    • If toggled OFF, you can customize these fields to appear differently in Ticket View mode.

    • We Recommend turning this feature ON so labels on the Ticket View screen will match when in Ticket Edit mode.

Unit Price and Discount Editor Options

The highlighted areas show changes made to the following label settings when applying a price override or discount to a line item.

  • Label for ‘After Tax’ (Tax-Inclusive) Unit Price checkbox when editing Unit Price… - After Tax → After GST.

    • This is used when overriding the unit price for a single item.

    • By default, the “After Tax” selection will be checked when you adjust an amount.

    • e.g. If you adjust an item to $10 (with a GST Tax Rate of 10%), the item’s price is $9.09 + $0.91 tax.

  • Label for Effective “After Tax” (Tax-Inclusive) Discount when editing item Discount… - After Tax → After GST

    • This is used when applying a discount for a single item.

    • By default, the “After Tax” selection will be checked when you adjust an amount.

    • e.g. If you discount an $11 item (with a GST Tax Rate of 10%) by $1, the item’s price is $9.09 + $0.91 tax, for a line total of $10.
