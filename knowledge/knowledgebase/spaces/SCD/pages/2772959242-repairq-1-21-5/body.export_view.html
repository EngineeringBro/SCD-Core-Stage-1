New Features

We’ve improved system stability for importing larger files by moving the process to an async queue. This allows data processing tasks to run in the background, reducing system load, minimizing delays, and ensuring a smoother user experience.

  • Background Processing: Users can leave the page after submitting an import while the file is processed in the background.

  • Completion Notification: A message banner will notify users when the import is complete.

  • Email Alerts: The user who submitted the import will receive an email notification once the process is finished.

Bug Fixes

  • We fixed an issue where importing a Catalog Item template would show the status as 'Scheduled' in the Import Queue but fail to insert the records. The import process will now correctly add or update the records.

  • We fixed an issue where adding a partial cash payment incorrectly set the ticket balance to $0. Now, only the entered payment amount will be applied, ensuring the remaining balance is correctly reflected.

  • We resolved an issue where users applying a payment to a ticket received ‘$NaN’ (Not a Number) in the UI when calculating the ‘Change Due’ amount, leading to invalid payment submissions.
