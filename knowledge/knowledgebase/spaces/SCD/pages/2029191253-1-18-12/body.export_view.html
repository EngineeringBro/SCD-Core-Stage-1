Released 10/31/2022 at 11:30pm CT

Minor Changes

  • Safeware bundles can now be removed, as long as the item eligible for the warranty is also removed.

  • A new Tip Amount column has been added to the Cash Reconciliation Report's Transaction Tab and Gateway Tab.

  • Made improvements to the Waiting for Payment display for processing payments.

  • Payment devices now unpair automatically if RepairQ sees duplicate IDs being used .

  • Quick Sale Optimizations settings have been added, allowing for fewer clicks and faster transactions on Quick Sale tickets.

  • Updated the Square integration so that the RepairQ Ticket ID will appear as part of the transaction description in the Square merchant dashboard.

Bug Fixes

  • Internal backorders, which had been experiencing issues when multiple purchase orders were involved, should now be calculating correctly.

  • Resolved a recent error caused by trying to include or exclude items within a service program's compatibility settings.

  • Fixed a bug that prevented the ticket status buttons from working on some Quick Sale tickets.

  • Removed an “All Locations” filter option from Appointments List Views that was causing problems.

  • Fixed an issue where Global Admins were not able to search for tickets at locations other than the one they were logged in to.

  • Previously released as a hotfix: Fixed a problem with the Square integration that allowed the connection authorization to expire, disconnecting Square from RepairQ unintentionally.

  • Fixed a problem with the Appointment Summary screen where the action button would appear as a blank blue square, preventing the user from taking action.

  • Fixed an issue on Custom Forms where creating a “When” condition caused errors when submitting the form.

  • Previously released as a hotfix: In some cases, duplicate Square transactions were being recorded on tickets. We updated how the integration works, to help prevent this situation.
