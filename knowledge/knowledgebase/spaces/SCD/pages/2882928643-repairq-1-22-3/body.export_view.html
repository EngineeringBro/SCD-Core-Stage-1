UI Fixes

  • Resolved a UI alignment issue where interactive elements, such as buttons, were either misaligned or had reduced clickable areas.

  • Updated the font style across all Qsite pages for improved readability.

Appointments and Customers

  • Resolved an issue where applying a keyword search on the Appointments page would result in an error, requiring users to clear the cache to re-access the page. Now, performing a keyword search will return relevant results as expected, without error or need for cache clearing.

  • Fixed an issue in Appointment General Settings where changes could not be saved if the "Enable Sale Request Type" setting was disabled without a selection in the dropdown. Previously, users encountered a requirement error on the dropdown even when the setting was disabled, blocking any changes from being saved. Now, the page saves as expected without requiring a selection in the disabled field.

  • Customer Profiles support embedding links to publicly accessible images in Customer Notes, such as image URLs hosted on public websites. Please note that Google Drive, Dropbox, and direct image uploads are not supported at this time.

Cash Rounding Enhancements and Fixes

  • Updated the label “Australian Cash Rounding to “Cash Rounding: Nearest 5¢” for clarity.

  • Removed the Cash Rounding settings: Half Up, Half Down, Half Even, and Half Odd. These rounding methods, typically used in financial reporting and accounting, were not required for retail settings.

  • Resolved an issue where users were unable to complete a return/refund for an inventoried item if Tax-inclusive and Cash Rounding settings were active.

  • Resolved an issue where users were unable to perform partial returns on tickets with multiple line items when Cash Counding was applied to the total amount. Previously, the "Save Ticket" button would remain disabled after adding a refund transaction.

Tickets and Custom Forms

  • Resolved an issue where editing an item on a trade-in ticket with multiple items caused it to be replaced by the last item added. Items now retain their original details when edited and saved.

  • Resolved an issue where completing a Custom Form and then saving the ticket could cause an Error 500.

  • Resolved an issue where creating a new Custom Form field without completing required fields (Label, Order, etc.) would result in the form appearing to save but loses all progress instead. RepairQ now properly displays a warning when required fields are blank and prevents saving the new field until all required fields are completed.

Inventory Transfers

  • Resolved an issue where the cost of inventory items was incorrectly overridden during an internal transfer, even when the setting for “Enforce Catalog Item Price” was disabled. Now, the item's existing cost will remain unchanged unless “Enforce Catalog Item Price” is explicitly enabled.

Invoicing

  • We’ve fixed an issue where the “Enable Wholesale Pricing” setting incorrectly loaded the retail price instead of the cost price when the associated Billing Agent was linked to a ticket. Items will now correctly load with the cost price, removing the need for manual adjustments.

Imports

  • Resolved an issue with Customer Imports where errors in validating the Primary Phone Number and Referral Customer fields prevented a successful import.
