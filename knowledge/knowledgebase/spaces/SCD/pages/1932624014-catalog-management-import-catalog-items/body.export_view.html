RepairQ offers convenient inventory management by allowing users to add and update Catalog Items via an import file. Users can effortlessly create catalog items, registering the items and services available in your store. This process is streamlined through import files, supporting .xlsx (Excel), .csv (comma-separated values), and .txt (tab-separated values), with .xlsx recommended for the best experience.

  • Efficient Inventory Management - Easily add, modify, and maintain items in your Catalog.

  • Streamlined Data Entry - Quickly upload items in bulk, reducing errors and saving time.

  • Flexibility - Keep up with the latest devices and services as they become available.

Columns are matched on the header name, so you must have the correct header names (provided in the template) to import the column values successfully. Any unrecognized column headers will be ignored; only the values you wish to add or update need to be present in the import file. If no matches are found for a record in your import file, RepairQ will create a new catalog item record from the import row.

Downloading the Catalog Import Template

On the Navigation Bar, go to Settings → Catalog. When viewing the Catalog List, click the Options button near the top right of the page. This will expand the list of available actions. From the list, click Import.

An import section will appear under the Catalog Item List. Under the Import Catalog Items header, click Blank Catalog Import Template to download an Excel file, with column names, ready for entering catalog data.

  • Catalog Import Template - This will download an Excel file with the current List View results of your catalog items, ready for changes. We recommend filtering your catalog to more precise results to reduce wait time during export and import. This is ideal for scoping results to the items you need to make changes to.

  • Blank Catalog Import Template - This will download a blank Excel file with column names, ready for new data.

After downloading the blank template, open the file and add items to add to your catalog, then save the document. The following columns require data to create a new catalog item in RepairQ: Item Type, Manufacturer, Name, and Active.

Required and Suggested Fields

Required values for creating a catalog item are Item Type, Manufacturer, Name, and Active. See our article, Catalog Item Import: Rules and Fields, for additional behaviors and category details for importing items.

  • Item Type - Assign a classification for this catalog item. This must match an existing Item Type in RepairQ.

  • Manufacturer - Assign the manufacturer of this catalog item. This must match an existing Manufacturer in RepairQ.

  • Name - This is the primary name of the Catalog Item.

  • Active - Active Catalog Items are searchable and can be added to tickets.

    • Use the number one “1” to mark the item as active.

    • Use the number zero “0” to mark the item as inactive. This is used when retiring an item from use. An inactive catalog item cannot be added to a ticket and will be removed from the default Catalog List view. You can still locate and reactivate items by filtering for Attibrute = Inactive.

  • Repairable - Items marked as repairable can be added to a Repair Ticket as the Repair Device.

    • Use the number one “1” to mark an item as repairable.

    • Use the number zero “0” to mark an item as non-repairable.

  • Inventoried - Items marked as inventoried have a quantity that can be tracked in RepairQ. These items must be stocked to be used on tickets and can be replenished via Purchase Orders, inventory adjustments, and Inventory Counts. Non-inventoried items can be added to tickets however, there is no way to track the amount of this item in your possession.

    • Use the number one “1” to mark an item as inventoried. Once this item is added to a ticket or purchase order, this field will be locked for editing to maintain data integrity.

    • Use the number zero “0” to mark an item as non-inventoried. This is typically used for labor, services, and other items that do not require a physical quantity on a ticket. Once this item is added to a ticket, this field will be locked for editing.

  • Serialized - Items marked as serialized will require a serial number when the item is added to your inventory and added to a ticket.

    • Use the number one “1” to mark an item as serialized. Only items marked as inventoried can be marked as serialized. Once this item has been added to a ticket or purchase order, this field will be locked for editing.

    • Use the number zero “0” to mark the item as non-serialized. Once this item has been added to a ticket or purchase order, this field will be locked for editing.

  • Price - This is the retail price of the item. This is the amount that the customer pays to purchase the item. This may be affected by factors like profit margin, market demand, etc.

    • If a price is entered for the Catalog Item, it will automatically be applied when adding the item to a ticket or into inventory.

  • Cost - This is the expense of your business to produce, acquire, or deliver that item. This may be affected by factors like your purchase price of the item, materials, labor, overhead, and other associated costs.

    • If a cost is entered for the Catalog Item, it will automatically be applied when adding an item to your inventory.

Import the Catalog Template

When you have saved the file and are ready to upload your import, click Choose File and select your file from the file explorer. Once the file is selected, click Import. RepairQ will begin processing your import file. You will be notified via email when the import is complete, and you can check the Import Queue for any additional details.
