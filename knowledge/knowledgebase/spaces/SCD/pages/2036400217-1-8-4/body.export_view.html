Released on: 9/02/16 10:58 CT

This is a minor release with a few bug fixes.

Ticket Custom Forms

The custom forms "required when" setting was not functioning properly since RepairQ 1.8.3. This update restores the enforcement of custom form requirements.

Inventory Import

We changed the inventory import template column header "Catalog Item" to "Name" so that it matches the expected column headers expected for importing.

Report Exports

Several report exports were cutting off number values that contained commas. Any numbers trailing commas were dropped from the value. This issue has been resolved.

Ticket Import

Adding services to a ticket with an import was not working properly due to the header column names not mapping to the import file 

Listen360 Integration

Updated ticket export format from .xlsx to .csv.
