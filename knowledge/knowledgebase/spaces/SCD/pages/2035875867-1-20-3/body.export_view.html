Released on 8/29/2023

Fixes/Improvements

  • Resolved an error caused when changing the total of available results in List Views.

  • Resolved an error caused when saving and submitting certain changes.

  • Resolved an error caused when accessing Validation Rules.

  • Fixed an issue that limited users from accessing the Invoice Aging Report.

  • Visual stability when using filters and pop-ups.

  • Various security fixes and improvements.

New Features/Enhancements

  • Staff Username and Password verification settings - New settings for required credentials for user timecard management, switching staff profiles within the program, and increased password security.

  • Save a Ticket as a Quote - Allow users to save any ticket as a quote, rather than leaving the ticket to create a separate quote.

  • Zip Code Validation - When a user creates or edits a Customer Profile, the system will perform a validation check for the country/zip. This is available to customers in the US, Canada, GB, and Australia.

  • The option for Part Recovery has been added to Catalog Quick Create.

  • Fixed an inaccurate error description when using a debit-only card for an integrated payment.

  • Updated the credential field for PIN/Password on the login screen.

  • Optimized inventory count process when assigning items with 0 quantity in stock

Qsite Improvements and Fixes

  • New architecture which improves accessibility and compliance.

  • Fixed Qsite settings affecting breadcrumb visibility.

  • Fixed an issue where a compatible Service was added as the Repair Device when submitting from Qsite.

  • Fixed an issue affecting time slot visibility when using Qsite scheduling for same-day appointments.

  • Qsite email validation allows existing customers to enter a different email address than what is on record. This will note the ticket that they would like to be contacted at an alternate email address.

  • Qsite will bypass any custom device validation rules set by an Admin. This allows customers to complete the quote/appointment process. Staff will be required to update the validation fields when editing the ticket.
