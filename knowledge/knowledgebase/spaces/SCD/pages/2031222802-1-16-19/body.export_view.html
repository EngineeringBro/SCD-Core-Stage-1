Release 1.16.19 is a minor release which adds a handful of new features.

Holiday Settings

Holiday settings for a location can now be managed by the Location Admin role. Prior to this, only an admin could manage holiday settings for all locations

Automated Notifications

We added a few new short codes for automated notifications, and we added a new “reply to” input where you can customize the reply to address for emails.

New Short Codes for message content

  • {customer_pri_phone}

  • {customer_email}

  • {devices} shortcode will now return suggested device info if it exists

  • {device_problem_description} will return the ticket device problem description / diagnostic field

Reply To

You can define a reply to address using the same shortcode options as the send to address. If you leave this input blank, it will default to your current reply to address.

Catalog Price Advertised

There is a new “price advertised” catalog item attribute which will allow you to choose whether or not the price for an item will be shown in Qsite. If you unset the price advertised attribute, then Qsite will treat the item the same way as the “price unavailable contact us for more info” scenario. You can set this attribute for the global catalog, and in location overrides. The attribute will also be returned as the value “is_price_advertised“ for catalog items returned in our category and catalog APIs.

Announcements

We added the ability to manage announcements at the company level. You can edit and delete an announcements across multiple locations from a single interface via the company settings announcements page. 

Qsite Ticket Lookup

There is a new Qsite screen /qsite/lookup where you can enter a ticket number or claim number to trigger a secure link to view the ticket details, and manage an attached appointment.
