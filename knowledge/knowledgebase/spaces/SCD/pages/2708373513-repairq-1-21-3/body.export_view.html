Enhancements and Bug Fixes

  • When swapping locations or logging in via the timeout popup, you will now continue on the page you left off, rather than being redirected to the homepage.

  • The Ticket Approval ‘Thumbs Up’ button has been fixed, and the status display issue, which previously showed HTML code instead of the actual status, has been resolved.

  • When transferring inventory to fulfill a backorder item, the cost from the original location is retained, and the placeholder (backorder) item is updated with the correct inventory details from the transferred item, ensuring consistent cost management.

  • The ‘Ready For Shipment’ status has been removed from the ticket return process.

  • Resolved an issue where users received an error when manually editing un-batched RMAs from 'RMA Sent' to a reconciled status.

  • Resolved an issue where HTML formatting in Ticket Notes could cause certain elements of the ticket to disappear while in Edit mode.

  • Resolved an issue where some users experienced a '500 Internal Server Error' when creating a new Catalog Item.

  • Resolved an issue that caused totals to be hidden on receipts when printing while the ticket was in ‘Waiting for Payment’ status.

  • Resolved an issue where clicking ‘Unselect All’ in the Staff List filter failed to clear all selected home locations.

  • When the Parent Item of a bundle is removed, only the child items will be removed. Other ticket items below it (that are not part of the bundle) will remain intact.

  • Resolved an issue where the cost and price of an inventory item were incorrectly reset to the catalog default when the item was edited by a user. When editing an inventory item, the previously entered cost and price will be retained.

  • On the Ticket General Settings page, both the ‘Select All’ and ‘Unselect All’ options are now functioning properly for all applicable fields.

  • Resolved an issue where Ticket Statuses became unavailable for selection if the required Ticket Resolution/Ticket Assignee modal was canceled by the user. (Company Settings → Tickets → General Settings → Status Advancement Requirements)

  • In the Tax Report, we removed the visible code next to the ‘Payment Plan’ filter.

  • The following issues have been fixed for the Staff List filter:

    • The results now display a single row per user, regardless of their location assignments. Previously, results returned multiple rows for the same user if they were assigned to more than one location.

    • When the Name filter is left blank, all applicable, active users will be returned in search results. Previously, when no specific name was entered in the field, the results returned a single user in the results.

  • Resolved an issue where store-level users could only see other users within their location if those users were assigned to a Home Location. Now, all users associated with the location will appear, regardless of their home location assignment.

  • Users can now reorder Form Fields in Custom Forms, and the selected order will be accurately reflected on tickets and in printed forms, resolving the issue where fields were displayed in the order of creation.

  • Resolved a readability issue on the Sales Tax Report where the text of selected items in dropdown fields overlapped with the dropdown arrow.

  • Resolved an issue where dropdown filter selections on the Tax Report page were sometimes unresponsive.

  • Resolved an issue where filtering the Catalog List yielded fewer results than expected or resulted in an error.

  • Fixed an issue where changing the Country field from ‘United States’ to a different country incorrectly retained the State field validation, preventing the user from creating a new Customer Profile.
