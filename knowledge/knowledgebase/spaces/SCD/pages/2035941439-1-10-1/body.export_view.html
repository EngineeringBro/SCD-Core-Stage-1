Released 01/25/2017 at 9:04 PM UTC / 3:04 PM CT 

This is a minor update to address bugs and add minor features.

Integrated Printing

Ticket Invoices

We resolved a few issues related to layout and fonts for integrated prints.

Ticket Receipts

We resolved a few issues related to layout for integrated prints.

Manual print menu (on the ticket view)

The print menu will now only load printers that you have defined in RepairQ, rather than loading all printers installed on your system. We implemented this change to prevent printer clutter in your print menus, and because you will have much better results if you add your printer settings into RepairQ.

In addition, the print menu will now show the fallback option to "print with the browser dialog" without any delay when you do not have the QZ Tray application running.

Print Status

The print status window now properly reports connection errors when the QZ Tray application is not running.

Printer Settings

We added some inline help popovers to the printer settings page to help everyone with configuration.

Non-integrated printing

For your convenience, we restored the "print" button to all print pages, and all print pages will open the print dialog on page load.
