1.17.10 is a minor update including some small features and bug fixes.

Released 11:30 CST on July 20

New Features

  • Require a ticket resolution before entering a specific ticket status.

  • Require a ticket assignee before entering a specific ticket status.

Enhancements and Bug Fixes

  • A VCT is now required when voiding a ticket so that any voided payments can be accounted for when closing the cash drawer.

  • Added new in-app controls to manage RepairQ support team login access to perform troubleshooting.

  • The inventory item list (for a given catalog item) will now show the location specific price for the catalog item instead of the default catalog item price.

  • The device list manufacturer filter will now show only the manufacturers that have repairable devices linked.

  • Performance optimizations that will reduce page load times.

  • We fixed an issue with Qsite that would limit the number of compatible services that would be listed for a device.

  • Fixed an issue with saving IP whitelist changes for a staff user.

  • Fixed an issue with including custom forms on claim tickets.

  • Fixed an issue that prevented unsetting the billing agent on the ticket.

  • Fixed an issue that would hide the “quick links” on the dashboard when the check-in widget was enabled.
