Admins can customize user verification settings when staff clocks in/out of the system and when switching User Profiles within the application. We’ve also included additional password security settings when a staff member creates or changes their PIN/Password and auto-deactivation for profiles with no System Log activity.

  • Quick Switch Settings - Admins can allow/restrict the ability for Staff to switch between profiles without logging out of their current location, allowing for time-saving changes while managing multiple users on a terminal.

  • Timecard Verification Settings - Admins can change whether the user must type in their Username and PIN/Password or only their PIN/Password to clock in/out of the system.

  • Password Security - Admins can require users to establish a unique, strong password for added security.

  • Auto-Deactivation for Inactive Staff Profiles - Automatically deactivate staff after a set number of inactive days.

Note: We recommend using RepairQ in a single tab/window since switching users only updates your current page/tab. Any additional screens/tabs not updated with new information can create issues when submitting data.

Access Staff Username and Password Settings

Log into your environment as an Admin. From the Navigation Bar, go to Settings → Company Settings. On the Settings Overview page, scroll down until you see the Miscellaneous category. From the available options, click Miscellaneous.

Staff Management Settings

Manage automatic termination of inactive Staff Profiles, switching profiles, and Password/PIN security requirements.

  • Mark Staff “Terminated/Not Employed” after N days of activity - Automatically deactivate staff after a set number of inactive days. Users can no longer access the environment if they show no System Log activity based on the Total Days assigned.

  • Enable Fast User Switching - Activate or deactivate the Switch User option from the Navigation Bar. When active, this allows fast switching between users if a user is already logged into the system.

  • Allow PIN-only user switch and timeclock - When selected, staff will be prompted to enter only their PIN/Password to switch users or when clocking in/out of the system. When unchecked, the user must enter their Username and PIN/Password.

  • Show optional Username field with PIN-only - When selected, this setting works with the above feature to show both the optional Username and required PIN/Password fields when switching users or clocking in/out. The user can enter both fields or leave the Username field blank and only enter their PIN/Password.

  • Require unique PIN - When activated, users must create a unique PIN/Password when establishing login credentials. When inactive, staff members can use a PIN/Password that may match another user’s password.

    • If you leave this feature deactivated, we recommend disabling “Allow PIN-only user switch and timeclock” to avoid conflicts if a user enters an identical PIN/Password to another user.

  • Require strong PIN/Password - Enforce a stronger format for user passwords.

    • Passwords must include at least 6 characters, contain at least one uppercase letter and one lowercase letter, and must contain at least one symbol (~!@#$%^&*()-_=+[{]}\|.;:‘“,.<>/?).
