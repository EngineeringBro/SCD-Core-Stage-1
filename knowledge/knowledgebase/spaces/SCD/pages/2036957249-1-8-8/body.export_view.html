Release Launched 9/29/16 4:11PM CT

This is a minor release with a few bug fixes.

Inventory Count - Reconcile All Zero Variance

The "Reconcile All" feature on the count screen marks all counts with a zero variance as "reconciled." No inventory records are adjusted when using the reconcile all feature, and only the count record status is updated from "counted" to "reconciled." An issue was identified with serialized item counts where it was not required to have all matching serials before a count could be marked as reconciled, therefore potentially missing some count records where the counted qty matched the instock qty, but the serial numbers did not all match instock serials. This has been updated, so that in the event that this occurs, the count record is skipped, and you are required to handle the reconciliation of the count directly. In this way, counts with mismatched serials will not be marked as reconciled without handling the serial number discrepancies.

Ticket Item Damaged/Defective Display Label

There were a few scenarios where the ticket item status would show "damaged" when it should show "defective." No accounting records are affected by this update; only the status displayed on ticket screens is affected by this update.
