RepairQ uses QZ Tray integration for one-click receipt, invoice, and label printing without navigating additional print dialog screens. Users can further streamline their ticket workflow by creating customized triggers that automatically print receipts, invoices, and labels when progressing tickets when entering or leaving a specific ticket status.

  • Effortless Printing - Skip the hassle of additional print screens and save time by generating receipts, invoices, and labels through a single-click process.

  • Streamlined Workflow - Automated Printing saves time and effort by reducing manual printing tasks and creating a smoother experience for the user and customer.

  • Personalized Print Automation - Customize automation by directing different document types to specific printers (receipt, desk, label), triggered by ticket actions.

Integrated Printing offers a wide variety of new setups, configurations, and automation. We have included common troubleshooting tips and workarounds if you encounter any issues with QZ Integrated Printing. Please make sure your computer meets the QZ Tray Minimum Requirements before setting up the integration. If you continue to experience issues with QZ, please create a support ticket and provide the following information to our Support Team:

  • A detailed description of the issue

  • Brand and model of the printer you are experiencing the issue

  • Paper size being used

  • Your current operating system and version

  • The Current version of QZ installed on your computer

  • The current version of Java installed on your computer

  • Additional technical information, photos, videos, etc. will assist in troubleshooting your issue.

Workaround - Bypass Integrated Printing and Print Manually (Print Dialog)

If Integrated Printing is not working correctly, you can still print using the browser dialog. While viewing a ticket, the Ticket Action Bar is viewable on the right side of the page. There are multiple icons based on what you are trying to print — Receipt, Invoice, or Label. When viewing the ticket you would like to print, click the dropdown arrow next to the print icon to expand additional options. From the list, select Print With Browser Dialog.

This will open the print job in a new window. Select your printer destination and layout, then click Print.

Troubleshooting: Updating Your Printer Settings

If you encounter alignment or blank page issues, you can try updating the margins and dimensions of your paper. Navigate to Settings → Location Settings → Integrated Printing → Printers.

We recommend using the Metric (millimeter) settings for most Star and Epson printers.

If your receipt printer appears to print but only prints out a one-inch blank receipt:

  • Check your DPI setting in RepairQ. Make sure it is converted from Dot Per Inch to Dot Per Millimeter if using Metric.

    • 1 Dot per inch = 0.0393701 Dots per mm

    • 203 DPI = 7.9921303 (~8) DPM

    • 300 DPI = 11.81103 (~12) DPM

    • 600 DPI = 23.62206 (~24) DPM

If you receive a "Cannot call method public void…" error:

  • Check your DPI setting in RepairQ. Make sure it is converted from Dot Per inch to Dot Per Millimeter if using Metric.

  • Close the QZ Tray application and reboot your computer. Re-open the application and try again.

  • Download and install the latest version of Java. Check to make sure which version of Windows you are running (32-bit vs 64-bit). Check for the appropriate version of Java.

    • Uninstall any older versions of Java when it prompts you.

  • If you still see the error, try launching the QZ Tray app from the command prompt. This allows you to see error output from QZ and/or start QZ Tray with more memory (The default is 512M. Try 1024M).

  • Close the QZ Tray application.

    • Open "Command Prompt"

    • Copy and paste this exact command and hit Enter on your keyboard.

    • java -Xms512M -jar "%PROGRAMFILES%\QZ Tray\qz-tray.jar"

    • Re-open the QZ application.

If you receive the message: "Error: connection closed before response received":

  • Check your DPI. Make sure it is converted from Dot Per Inch to Dot Per Millimeter if using Metric.

  • Close down the QZ Tray, reboot your computer, and then re-open the application.

  • Make sure Java is updated. If prompted, uninstall any older versions of Java.

  • Clear your browser cache.

Troubleshooting: QZ Software

Currently, RepairQ is updated for version 2.2.1 or older. Newer versions of QZ have not been tested for compatibility.

Restart the QZ Tray or Redownload QZ Software

  • Close the QZ Tray program, reboot your computer, and then re-open the application. make sure that QZ Tray is set to automatically start.

  • Ignore the QZ Tray program in any security software you have running - Avast, Norton, etc.

  • If you have installed more than one copy of QZ Tray, uninstall any QZ programs and reinstall the correct version.

    • Having more than one application installation (QZ and QZ (1)) can cause miscommunication.

    • If in doubt, uninstall any programs related to QZ, re-download, and install.

    • Download QZ Tray here

    • Reboot your computer, make sure QZ is running in your system tray, and try printing again.

Downgrade to a Previous Version of QZ

Sometimes, an older version of QZ works better than the latest updates.

  • Uninstall QZ from your machine and then reboot.

  • Download a previous version of QZ from the QZ Archive page.

  • Install 2.2.1 and open the QZ Tray app, then try printing again.

If QZ Tray Randomly Disconnects or Closes/Crashes

Make sure you have the minimum recommended 6 GB RAM.

  • Close down the QZ Tray app and reopen the application.

  • Uninstall the QZ Tray app and reinstall it.

  • Disable any antivirus software or allow the QZ Tray app to bypass it.

  • Reboot your computer.

  • Adjust your Paper Height to a lower setting from ~78 inches (2000 mm) to ~20 inches (500 mm).

Troubleshooting: Install the latest version of Java on your computer

If everything appears to be set up properly, but the Print Job screen stays open and never completes, you can try the following:

  • Download and install the latest version of Java. Check to make sure which version of Windows you are running (32-bit vs 64-bit) and follow the instructions for downloading the appropriate version. Java Downloads for All Operating Systems.

  • Uninstall any older versions of Java when it prompts you to.

  • Reboot your computer.

Cash Drawer is Connected to Your Receipt Printer but Does Not Trigger When Printing a Receipt

  • Make sure no third-party print utilities (usually installed with the printer) are interfering with the print. Sometimes those utilities override the print jobs with their default settings and need to either be disabled or adjusted accordingly.

  • The communication between different applications and the driver seems to be the common variable in this problem.

Clear your Browser Cache

On Chrome Browser, navigate to Options → Settings.

On the left Settings column, click Privacy and Security.

Click on Clear browsing data, and a pop-up will appear. Under the Advanced tab, select Cached images and files (uncheck any additional fields) and click Clear data. Close the browser and restart the application.
