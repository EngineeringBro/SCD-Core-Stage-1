RepairQ's import feature enables users to batch-add records, streamlining data management and reducing the need for manual entry. For ticket management, this feature allows users to import and create new tickets in bulk, which is especially useful for businesses handling large quantities of sales or repair tickets.

Users with Admin, Manager, or Inventory Manager roles can take advantage of built-in import templates to manage various records, such as Tickets, Customers, Purchase Orders, and Inventory. Imported tickets are added in a "New" status, making it easy to track and manage them from start to finish.

Before importing tickets, it’s important that your Catalog and Customer List are established. This ensures that all relevant products and customer information are available in the system, allowing tickets to be accurately matched and recorded. Currently, only non-inventoried items can be added on a ticket import.

Note: For users migrating data from another software system, we recommend exploring one of our onboarding options to facilitate a smooth and comprehensive transition.

Import Guidelines

Columns are matched on the header name, so you must have the correct header names (provided in the template) to import the column values successfully. Any unrecognized column headers will be ignored; only the values you wish to add or update need to be present in the import file.

Download the Ticket Import Template

Before importing tickets, it’s important to import your Catalog and Customer records first. This ensures that all relevant products and customer information are available in the system, allowing tickets to be accurately matched and recorded.

On the Navigation Bar, go to Tickets. When viewing the Ticket List, click the Options Button near the top right of the page. This will expand the list of available actions. From the list, click Import.

An “Import Tickets” section will appear under the Ticket List. Click the Ticket Import Template dropdown to select and download an Excel file with pre-defined column names. This file is formatted specifically for entering ticket data, ensuring it aligns with our system's requirements. You can choose between two templates:

  • Claim/Repair Ticket Template - Use this template for importing repair or service-related tickets.

  • Sale Ticket Template - Use this template for importing sales tickets.

Sale Ticket Import

After downloading the blank Sale Ticket template, open the file and add items to add to the document. The following columns require data to create a new ticket in RepairQ: Type, Location, and Note. After adding the data, be sure to Save your changes.

Note: You cannot replace or modify existing tickets by importing new ones. Each import will create a separate batch of new tickets.

Required and Suggested Fields

  • Type (Required) - When importing Sales Tickets, make sure each entry in this column is listed as "Sale" for each row to avoid errors during import.

  • Status - Assign each ticket an open status, if applicable. If left blank, tickets will automatically be added with a "New" status.

    • The Status field accepts any ticket status except Void and Closed, though we recommend using “New” for easy tracking.

    • You can view and manage available ticket statuses in Company Settings → Tickets → Ticket Statuses. Assigning the correct status helps ensure tickets enter the appropriate workflow stage.

  • Location (Required) - Specify the Location (Short Name) of the store where the ticket will be created.

    • This can be found in Company Settings → Locations. Entering the correct short name assigns the ticket to the proper location.

  • Ticket Items - Enter the Device SKU for the specific product or device to be added to the ticket. Make sure the SKU aligns with your Catalog to accurately track repairs and sales.

    • Only non-inventoried items can be added to an imported ticket.

  • Notes (Required) - Include a note on each ticket explaining the purpose of the import. This helps provide context for the imported ticket entries.

  • Customer First Name - If linking this ticket to an existing customer, use this field to match the customer’s first name to an existing Customer Profile. If no exact match is found for both first and last names, a new Customer Profile will be created.

  • Customer Last Name - Similarly, use this field to match the customer’s last name to an existing Customer Profile. If a match for both first and last names isn’t found, a new Customer Profile will be created.

  • Customer Phone - Enter the customer’s phone number to further match this ticket to an existing Customer Profile. If no match is found, a new Customer Profile will be created.

Repair Ticket Import

After downloading the blank Repair Ticket template, open the file and add items to add to the document. The following columns require data to create a new ticket in RepairQ: Location (Short Name), Device SKU, Device Diagnostics, and Note. After adding the data, be sure to Save your changes.

Note: You cannot replace or modify existing tickets by importing new ones. Each import will create a separate batch of new tickets.

Required and Suggested Fields

  • Status - We recommend importing tickets in either “New” or “Waiting for Payment” status.

    • If left blank, the ticket will default to “New” status.

  • Location - Assign the Location (Short Name) for the store or branch where the ticket will be logged.
    This short name is listed under Company Settings → Locations.

    • Using the correct short name ensures that tickets are assigned to the appropriate location.

  • Customer (Recommended) - Each ticket should be linked to a customer using their unique Customer ID.

    • Navigate to the customer’s profile in RepairQ. The Customer ID is the numeric value at the end of the URL in the address bar. Entering the correct ID ensures that tickets are assigned to the respective customer and their repair or purchase history.

    • If you leave the Customer field blank, the ticket will be imported under the name: “Null, Null” and you must edit the profile with valid customer information before you can save the ticket again.

  • Billing Agent - If the ticket is associated with a Billing Agent, enter the corresponding Agent ID.

    • This ID can be found by navigating to Customers → Billing Agents and selecting the specific Billing Agent to view their details. Similar to the Customer field, Billing Agent ID is the numeric value at the end of the URL in the address bar.

  • Device SKU (Required) - The unique SKU assigned to the Repair Device in the Catalog.

    • The device must have an existing SKU assigned and be flagged as Repairable in the Catalog for it to be valid during the ticket import process.

  • Device Serial - The unique serial number that identifies the repair device. This number helps ensure the correct device is logged and tracked throughout the repair process.

  • Device Description - A detailed description of the repair device, including attributes such as color, memory/storage capacity, and any pre-existing blemishes or defects. This information helps document the device's condition before service.

  • Device Password - The password required to access the repair device, if applicable. This is often needed for diagnostic or service purposes.

  • Ticket Item SKU - The SKU for the specific Repair Service or Repair Item being added to the ticket. This could include labor services or replacement parts.

  • Device Diagnostics (Required) - The initial findings or reported issues with the device, as noted by the customer or technician. This information is crucial for understanding the repair requirements and providing accurate service.

  • Note (Required) - A general note added to the ticket. This can include customer instructions, technician observations, or any other relevant details for the repair process.

Importing the Ticket Template

When you have saved the file and are ready to upload your import, click Choose File and select your file from the file explorer. Once the file is selected, click Import. RepairQ will begin processing your import file. You will be notified via email when the import is complete, and you can check the Import Queue for any additional details.

Import Rules

Columns are matched on the header name, so you must have the correct header names (provided in the template) to import the column values successfully. Any unrecognized column headers will be ignored; only the values you wish to add or update need to be present in the import file.
