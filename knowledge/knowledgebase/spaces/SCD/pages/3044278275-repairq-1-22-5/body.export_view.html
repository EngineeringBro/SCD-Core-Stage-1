New Features and Enhancements

Settings

  • We've optimized the Company General Settings page to reduce load times in larger environments.

  • We've streamlined the Ticket Label Customization settings (Location Settings → Ticket General Settings → Customize Ticket Edit) to show only the relevant label options based on your selected “Default View.” This reduces confusion when customizing fields. If the Default View is set to:

    • Default - the the following fields are hidden: “Tax-Inclusive Unit Price (before discount)” and “Effective Quantity.”

    • Tax Inclusive - the following fields are hidden: “Unit Price (before tax or discount)” and “Effective Quantity.”

Purchase Orders and Catalog

  • We've modified the Internal PO process to remove automatic backordering. Now, when an Internal PO can't be fully fulfilled due to low stock, RepairQ can send a notification instead of creating backordered items. This change gives inventory managers better visibility into stock shortages without adding unnecessary backorders.

  • We've introduced new Automated Notification Triggers related to the Internal PO process:

    • Insufficient Stock for Internal Purchase Orders - This notification alerts you when there isn’t enough available stock to fulfill an Internal Purchase Order.

    • New Stock Available for Internal Purchase Orders - This notification triggers when new items are added outside of Purchase Orders or Imports—like through manual adjustments, inventory counts, or items returned from tickets. If there are existing Internal Purchase Orders or backorders for these items, RepairQ will notify you, ensuring you're aware that newly available stock can be used to fulfill open demands.

  • We've introduced an API integration that allows suppliers to send Purchase Order information directly to customer environments, streamlining the procurement process with seamless order submissions and reduced manual entry.

  • We've improved catalog management by automatically removing inactive, deleted, or merged items from service program catalogs and compatibility modals when they are no longer active in the catalog.

Bug Fixes

Settings and Exports

  • We've fixed an issue where users were prevented from saving to Company General Settings without selecting a rounding method, even when the Cash Rounding feature was inactive.

  • We've fixed an issue where exporting large files from the Tickets page would result in an error. Pagination has been optimized to reduce memory usage, allowing large-volume exports to complete successfully.

Inventory Management

  • We've resolved an issue where sent RMA records couldn't be searched due to the page continuously loading when applying filters for RMA Sent status. You can now filter and search these records without any issues.

  • Fixed an issue that prevented filtering the Inventory Transfer list by “Supplier.”

  • Fixed an issue that caused an error message when changing the List View Total on the RMA tracking page to either 50 or 100.

Tickets and Appointments

  • We've fixed an issue where serial numbers were not being saved when adding a device to a Trade-in Ticket.

  • We've fixed an issue where users were unable to edit appointments due to a "Concurrency Limit has been reached" error when updating fields unrelated to the appointment time.

Invoicing

  • We've fixed an issue where a Billing Agent couldn't be removed from a ticket once assigned. Now, you can clear the selection by choosing "Select Billing Agent" from the dropdown, and the change will save correctly.

  • We've fixed an issue in Invoice Template settings where the SKU field visibility toggle wasn’t working as expected. Now, when the SKU toggle is selected, the SKU will appear on invoices, and when deselected, it will be hidden as intended.

Attendance/Timesheets

  • We've fixed an issue in the System Log for Timesheet entries where changes to a user's timecard displayed "Timesheet (Error with timesheet relation)". The log now correctly shows the employee's name and details of the clocked hours that were changed.

  • We've fixed an issue when adding attendance records. The employee field was greyed out, preventing stores from adding new records. Admins and Managers can now select employees and log attendance as expected.
