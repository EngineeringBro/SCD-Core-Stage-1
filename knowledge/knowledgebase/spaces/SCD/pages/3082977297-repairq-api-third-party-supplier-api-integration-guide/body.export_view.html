This guide outlines how to integrate a supplier’s purchase order system with RepairQ using API. The integration enables seamless order submissions directly to the customer’s environment, streamlining the procurement process.

RepairQ/SCT Responsibilities

Our internal team will be responsible for the following:

  • Establishing an API integration agreement with the supplier.

  • Providing the supplier with a Postman API collection for purchase orders.

  • Supplying the supplier with an API key and a test environment for API call testing.

Supplier Responsibilities

Suppliers are responsible for the following:

  • Providing a settings page on their purchasing website where users can enable the RepairQ integration and fields to enter their Base API URL and API Key.

  • Once set up, when a customer places an order through the supplier’s system, the supplier will POST the order details to the customer’s RepairQ URL with the API Key as the X-API-Key header for authentication.

RepairQ User Responsibilities

Users are responsible for the following:

  • Creating a Supplier record in RepairQ

  • Generating an API Key for the Supplier record in RepairQ

  • Linking the Supplier SKUs to their catalog items. This mapping ensures that when an order is placed, each item can be accurately identified within RepairQ given the supplier’s SKU values.

  • Entering the necessary connection details into the supplier settings page:

    • Base API URL

    • API Key

Generate an API Key for a Supplier

Navigate to Settings → Company Settings. On the settings page, under the Catalog section, select Suppliers.

The Supplier Overview page displays a list of pre-populated and added suppliers. To generate an API Key, click the Generate API Key Button (cloud icon) next to the supplier. A popup will appear with details about the API integration. Click Generate API Key to create and view your key, along with additional integration details.

Note: If you supplier is not listed, you will need to create the supplier first.

Once you generate an API key, be sure to copy and save it for future use, as it cannot be retrieved again. Also, make sure to note the base API URL, which is required for the integration.

Note: Use the Copy Button to copy the key and paste this information in a secure place.

  • Base API URL - The unique URL specific to your environment, used as the endpoint for sending purchase order data. This is required to establish a connection between the supplier’s system and your database.

  • API Key - A secure, system-generated code used to authenticate the supplier’s requests when sending order data to your environment. It ensures only authorized systems can access and submit orders to your database.

Generate a New API Key

When you generate an API key for a supplier, a corresponding user is also added to your API User list. A user might need to reset the API Key for several reasons, including:

  • Security Concerns - If the API key has been compromised or shared with unauthorized parties, resetting it ensures continued secure access.

  • Access Management - When changing suppliers or if a supplier changes their system, a new API key may be needed to maintain proper integration.

  • Revoked Access - If a supplier no longer needs access or if the integration is being discontinued, resetting the key prevents further data exchange.

  • Troubleshooting - If there are problems with the API connection, generating a new key can help resolve authentication errors.

To issue a new API key, go to Settings → Company Settings. On the Settings Overview page, under the API Management section, select API Users.

In the API User list, click the Generate Key Button (key icon) next to the user you want to reset. A warning will appear notifying you that generating a new key will invalidate the existing one. Confirm to proceed and generate the new key.
