Item Types are high-level categories to organize your catalog. This allows you to assign specific attributes to the entire group of catalog items under these types. Item Types dictate assigned tax rates, price override allowances, identify whether items are parts or labor, establish parent/child relationships for bundling, and more. Since Item Types are groups of Catalog Items with similar characteristics, the characteristics designated for an item type apply to each Catalog item categorized under it. Before building your Catalog, it’s a good idea to decide what Item Types you need to set up first.

Some reports are organized by item types, so you should create one item type for each category of revenue you want to analyze separately. By default, there are a variety of item types created for use but you can also create new item types and set their characteristics upon creation.

Item Type Fields and Behaviors

Item Types allow you to set additional attributes for each category of items. When adding/editing types, you can manage the following elements and behaviors.

General

  • Name - This is the primary name to identify the Catalog Item category. This must be unique for each type.

  • Description - This is a brief explanation of the purpose/use of this Item Type.

  • Royalty Percent - If you have an agreement to pay a percentage of your revenue to the rights holder, you can set that amount here. This will apply to all items created under this category. This will override a royalty percentage set at the Location level.

  • Flag the Entire Ticket as Royalty Exempt - When this is selected, catalog items created under this category will exempt all items on the ticket from reporting a royalty percentage when added to a ticket.

  • Flag the Ticket Item as Royalty Exempt - When this is selected, catalog items created under this category will be exempt from reporting a royalty percentage when added to a ticket.

  • Can Override Price - When selected, users can override the Unit Price for these items on tickets.

  • Enforce Catalog Item price for Inventoried items - When selected, this will automatically enter the Catalog Item price for items added to inventory.

  • Disable Wholesale Pricing - This option exempts all items created under this category from wholesale pricing when added to tickets. For more info on Wholesale Pricing, see Invoicing: Creating Billing Agents.

  • Apply Use Tax - When selected, all items created from this category are subject to a Use Tax. This is referenced by the Use Tax that is assigned in your Location Settings.

  • Labor/Service? - Selecting this option designates this item as labor so it can be tracked in COGS and other reporting.

    • Allow Bundling (as Parent Item) - This flag is combined with the Part or Labor flag to allow the bundling of parts and labor into a single line item on a ticket. Your profitability report will also benefit from the extra information regarding the cost of goods for the services you render.

    • Labor/Service Type - Set where the labor price is charged by the hour or a fixed price.

      • Rate - The service price will stay consistent regardless of the Qty/Hours recorded on the ticket.

      • Fixed - Your service price will vary based on the Qty/Hours entered on a ticket.

  • Part - Selecting this option designates this item as a part so it can be tracked in COGS and other reporting.

    • Allow Bundling? (as parent item) - This flag is combined with the Part or Labor flag to allow the bundling of parts and labor into a single line item on a ticket. Your profitability report will also benefit from the extra information regarding the cost of goods for the services you render.

  • Allow Bundling under Part (as child) - This allows items created under this Item Type to be bundled under a part as the "child" item.

  • Allow Bundling under Labor (as child) - This allows items created under this Item Type to be bundled under a labor item as the "child" item.

  • Catalog Item Add/Edit Restrictions 

    • No restrictions -

    • Restrict catalog item management to the Admin role -

    • Restrict catalog item management to location-specific items, excluding Admin -

  • Priority - This allows you to assign a number from 0 - 100 to organize the list of item types. Lower numbers will appear higher on the list.

  • Active - This sets the item type as active in the catalog. Unchecking this box will mark the item type as inactive and hide any associated items from the catalog. This will also hide any associated inventory items but will not affect their inventory status.

Journal Entry Classification

This is used for the QuickBooks integration. Assigning journal entries at the Item Type level allows you to override the defaults set in your primary classifications. For more information, see QuickBooks Online Integration. Otherwise, these fields can be ignored.

Auto Generate Serial Numbers

In RepairQ, when a catalog item is marked as serialized, a serial number is required before the item can be counted and tracked in the system. The auto-serialization feature can generate a serial number for items when added/imported into inventory or when sending part cores back to the manufacturer. If you have inventory items that do not have a serial number, but you would like to assign one to them for tracking purposes, this will automate the process.

  • Enabled - This will enable/disable the Auto-generate feature.

  • Serial Prefix - (Optional) Add a unique prefix that will be part of the generated serial numbers.

  • Serial Format - Select the serial format when generating new serial numbers.

    • With Hyphen - Generates a serial number with a hyphen.

    • Without Hyphen - Generates a serial number without a hyphen.

    • UID Without Hyphen - Generates a unique identifier (UID) based on the Inventory ID for the item in RepairQ.
