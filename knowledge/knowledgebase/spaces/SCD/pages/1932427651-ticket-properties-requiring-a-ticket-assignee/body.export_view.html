The Ticket Properties feature allows users to add additional details to Repair tickets. Properties can be used for classification in reporting, filtering your Ticket Queue, as well as working with custom forms, notifications, and approvals based on assigned properties.

Admins can require a user to select an assignee before they can save a ticket in a new status. This is useful for reporting and quality assurance purposes. You can require an assignee for specific ticket types (Repair, Refurb, Claim) and which ticket statuses require a resolution.

For example, you can set a requirement for an Assignee on Repair tickets before progressing the ticket to In Repair status.

Require a Ticket Assignee

1. Log into RepairQ as an Admin. On the RepairQ Navigation Bar, go to Settings → Location Settings.

2. On the Settings Overview page, select Tickets from the list of options in the left column. From the dropdown list, click General Settings.

3. On the Ticket General Settings page, toggle ON the setting labeled Prompt for (and require) ticket assignee when advancing to the following statuses.

4. In the text boxes below, select which ticket status(es) and ticket type(s) will require an assignee. Once you have made your selections, click the Save button on the upper-right of the page to submit your changes.

Locating the Assignee Field of a Ticket

When creating a new Repair ticket or editing an existing Repair ticket, the Assignee Field is along the top row of the ticket. An assignee will be required before advancing the ticket to the configured status.

Note: The assignee dropdown will display active users from your Staff List.

If the Resolution field has not been selected, the user will be prompted to select an Assignee before progressing the ticket status.
